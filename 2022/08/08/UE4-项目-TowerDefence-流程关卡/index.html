UE4-项目-TowerDefence-流程关卡

发表于 2022-08-08 分类于 unreal engine

思路

  1. 创建新关卡，定义关卡游戏模式（绑定HUDClass）

  2. 在HUD.h文件中

    • 声明构造函数和重载BeginPlay()

    • 声明HUD中的HUD内容界面类（HUD内容界面中包含按钮之类的控件）

    • 以HUD内容界面类为父类创建子类Class

    • 声明HUD内容界面类的指针实例

  3. 在HUD.cpp文件中

    • 使用static ConstructorHelpers::FClassFinder<HUD内容界面名>类名(TEXT("路径/蓝图名")获取蓝图类中的资源

    • 将蓝图类中的资源.Class赋值给以主界面类为父类创建的子类Class

    • BeginPlay()函数中继承父类并且使用CreateWidget<父类名>(世界,以HUD内容界面类为父类创建的子类Class)

    • 将其添加到视口

  4. 按钮类.h

    • 声明在蓝图中出现的击中控件类

    • 使用UPROPERTY(meta = (BindWidget))实例化控件类

    • 重载函数NativeConstruct()

    • 声明函数void SelectLevel()

  5. 按钮类.cpp

    • NativeConstruct()中Super::继承父类

    • 给各个按钮使用OnClicked.AddDynamic绑定绑定函数

    • 定义函数

  6. HUD内容界面.h

    • 声明在主界面出现的控件蓝图中的控件类（按钮，自定义类，画布面板等）

    • 使用UPROPERTY(meta = (BindWidget))创建控件类的实例（与蓝图中的同名）

    • 重载NativeConstruct()

    • 声明函数InitSelectLevelButton()用于初始化选择关卡按钮

    • 声明函数ReturnMenu()用于返回主菜单按钮

    • 声明一个数组用于存储关卡按钮

  7. HUD内容界面.cpp

    • 定义NativeConstruct()调用InitSelectLevelButton()以及ReturnMenu()

    • 定义InitSelectLevelButton()函数

    • 定义ReturnMenu()函数（打开某关卡）

添加新界面到视口

知识点

ConstructorHelpers::FClassFinder<>()

用于获取某个蓝图的类型class

在使用它们的时候还得遵守如下规则：

  • 只能在类的构造函数中使用，如果在普通的逻辑代码中嵌套这份代码，会引起整个编译器的crash。（实际上里面代码就有检查是否在构造函数里，否则crash）

  • 其次，FObjectFinder/FClassFinder变量必须是static的，从而保证只有一份资源实例。

  1. **FObjectFinder**：一般用来加载非蓝图资源，比如StaticMesh、Material、SoundWave、ParticlesSystem、AnimSequence、SkeletalMesh等资源：
  1. **FClassFinder**：一般用来加载蓝图资源并获取蓝图Class。这是因为如果C++要用蓝图创建对象，必须先获取蓝图的Class，然后再通过Class生成蓝图对象：
  • FClassFinder的模版名不能直接写UBlueprint，例如：FClassFinder是错误的。创建蓝图时选择的是什么父类，则写对应的父类名，假如是Actor，那么要写成：FClassFinder，否则无法加载成功。
  • FClassFinder的模版名必须和TSubclassOf变量的模版名一致，当然也可使用UClass代替TSubclassOf。实际上TSubclassOf也是UClass，只是更加强调这个Class是从T派生出来的。
  • 在启动游戏时若报错提示找不到文件而崩溃（例如：Default property warnings and errors:Error: COD Constructor (MyGameMode): Failed to find /Game/MyProject/MyBP.MyBP）
    这是因为UE4资源路径的一个规范问题，解决办法有两种：
    1. 在copy reference出来的文件路径后面加_C，例如：”Blueprint’/Game/Blueprints/MyBP.MyBP_C’”（_C可以理解为获取Class的意思）。
    2. 去掉路径前缀，例如：”/Game/Blueprints/MyBP”

CreateWidget<>()

创建一个新的界面

大致步骤

  1. 创建新关卡命名为SelectLevel，重载游戏模式为TowersDefenceSelectLevelGameMode

  2. 在TowersDefenceSelectLevelGameMode.h中声明构造函数以及重载函数BeginePlay()

  3. 在TowersDefenceSelectLevelGameMode.cpp中定义构造函数，绑定HUDClass为ATowersDefenceSelectLevelHUD::StaticClass();，并定义空的BeginPlay()函数（记得包含头文件ATowersDefenceSelectLevelHUD）

  4. 在TowersDefenceSelectLevelHUD.h中添加构造函数与重载BeginPlay()函数；在最前方声明类UUI_SelectLevelMain；使用TSubclassOf以UUI_SelectLevelMain为父类创建一个名为SelectLevelMainClass的类；声明一个UUI_SelectLevelMain的实例SelectLevelMain

  5. 在TowersDefenceSelectLevelHUD.cpp中添加UI_SelectLevelMain的头文件，在构造函数中使用ConstructorHelpers::FClassFinder来获取SelectMain_BP蓝图类并且命名为SelectMain_BPClass，将SelectMain_BPClass.Class赋值给在.h文件中创建好的SelectLevelMainClass

  6. 在TowersDefenceSelectLevelHUD.cpp中的BeginPlay()中添加使用Super::继承一下父类，并且添加头文件#include "Blueprint/UserWidget.h"，然后使用CreateWidget<>()创建UUI_SelectLevelMain类的界面SelectLevelMain，并且使用AddToViewport()将其添加到视口上

  7. 在RuleofTheHUD.h中同理，声明Class UUI_MainScreen，声明构造函数，声明重载BeginPlay()函数，以UUI_MainScreen为父类创建MainScreenClass，声明UUI_MainScreen类型的指针实例*MainScreen

  8. 在RuleofTheHUD.cpp中同理，定义构造函数，使用static ConstructorHelpers::FClassFinder获取MainScreen_BP类，命名为GameMain_BPClass，并且将其.Class赋值给MainScreenClass；在BeginPlay()中使用CreateWidget创建新页面，并且使用AddToViewport()将其添加到视口

UPROPERTY(meta = (BindWidget))

以此宏声明开头的实例，需要在对应的控件蓝图中存在同名称，同类型的实例

则需要在以此C++类为父类的控件蓝图类中存在一个名为ReturnMenuButton的Button控件

UPanelSlot

指向UMG的排版样式

其中存在一个Content指向将要被排版的UMG

UGameplayStatics::OpenLevel()

打开关卡

大致步骤

  1. 分别在在以下位置创建

    • Content/UI/Select/SelectMain_BP

    • Content/UI/Game/MainScreen_BP

    • Content/UI/Select/ButtonLevel/LevelButton_BP

  2. 在图表->类设置->类选项中分别替换其父类为对应的C++类

  3. 在UI_SelectLevelMain.h文件中

    • 声明class UUI_LevelButton，class UButton，class UCanvasPanel

    • 以UPROPERTY(meta = (BindWidget))宏声明开头，声明UButton* ReturnMenuButton以及UCanvasPanel* SelectMap（画布面板）

    • 声明重载NativeConstruct()函数（后面不需要Override）

    • 声明函数void InitSelectLevelButton();（初始化选择按钮），void ReturnMenu();（返回菜单函数）

    • 创建一个名为AllLevelButton的数组，存储UUI_LevelButton*类型的变量

  4. 在UI_SelectLevelMain.cpp文件中

    • 定义NativeConstruct()函数

      • 使用Super::继承父类

      • 调用InitSelectLevelButton()

      • 实例ReturnMenuButton调用成员函数OnClicked通过AddDynamic()进行代理绑定UUI_SelectLevelMain中的ReturnMenu函数

    • 定义InitSelectLevelButton()函数

      • 使用if()判断是否为SelectMap画布面板

      • 是则创建一个UPanelSlot*类型的数组PanelSlotArray，其中存储SelectMap->GetSlots()

      • 使用for (UPanelSlot* PanelSlotEle : PanelSlotArray)遍历数组PanelSlotArray

      • 如果PanelSlotEle->Content转换为UUI_LevelButton等于UUI_LevelButton* LevelButtonArp

      • 则将LevelButtonArp使用Add()添加到AllLevelButton中去

    • 定义ReturnMenu()函数

      • 使用UGameplayStatics::OpenLevel()返回HallMap关卡
