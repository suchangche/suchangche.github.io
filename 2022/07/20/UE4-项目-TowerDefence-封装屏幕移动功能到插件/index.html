UE4-项目-TowerDefence-封装屏幕移动功能到插件

发表于 2022-07-20 更新于 2022-07-26 分类于 unreal engine

封装屏幕移动功能到插件

效果：鼠标移动到屏幕边缘就可以造成屏幕相应方向的移动

知识点

UE的屏幕坐标

UE4的屏幕默认左上角为(0,0)

IsNearlyEqual()

源码

bool IsNearlyEqual(float A, float B, float ErrorTolerance = SMALL_NUMBER)

含义

A-B，如果值在ErrorTolerance之间，则返回true

注意事项

处于FMath命名空间中，在之前需要先FMath::

例如：

PlayerController->GetViewportSize()

源码

含义

获取屏幕大小，并将其X，Y值传给SizeX和SizeX

注意

必须由PlayerController->调用

PlayerController->GetMousePosition()

源码

含义

获取鼠标位置并将其传给参数LocationX和LocationY

注意

依旧需要PlayerController->调用

bShowMouseCursor

含义

为true则显示鼠标；为false则不显示鼠标

AddActorWorldOffset()

源码

含义

根据输入的偏移值（FVector类型）来使得物体进行位置偏移

FInputModeGameAndUI

一种数据结构，可以设置游戏和UI的输入模式

SetInputMode()

设置输入模式，其参数可以为FInputModeGameAndUI类型的变量

思路

  1. 在屏幕设置一个长方形的边界，在此长方形之内，鼠标随意移动，一旦超出边界，则屏幕往相应位置移动

  1. 打开TowerDefence\Plugins\StoneToolClass\Source\StoneToolClass\Public\Tool\文件夹，创建ScreenMove.h,并且在相应的Private\Tool\文件夹下面创建ScreenMove.cpp文件

  2. 在ScreenMove.h引入必要的头文件

    • 添加一个枚举EScreenMoveUnits，用于标记鼠标的位置状态

    • 声明APlayerController类

    • 创建一个结构体FScreenMoveUnits,在其中声明一个以EScreenMoveUnits为返回值的函数CursorMove，形参是APlayerController的实例，并且在.CPP文件中创建该函数的定义

  3. 在ScreenMove.cpp中定义CursorMove函数

    • 判断PlayerController是否有效，如果为true，则

    • 定义屏幕尺寸SizeX和SizeY，初始值赋值为INDEX_NONE

    • 定义鼠标位置MousePostionX和MousePostionY，赋值为-107374176.f

    • 通过APlayerController->GetViewportSize()获取屏幕尺寸并且传参给SizeX和SizeY

    • 通过PlayerController->GetMousePosition()获取鼠标位置并且传参给MousePostionX，MousePostionY

    • 用if()来限定MousePostionX，MousePostionX的取值范围（<=SizeX和SizeY&&>=0）

    • 通过FMath::IsNearlyEqual()来修改EScreenMoveUnits的状态，分别修改为上下左右，左上左下，右上右下；如果以上状态均不是，则返回为Screen_None

  4. 在ScreenMove.h中声明MoveDirection函数实现目标的移动

    • 返回值为bool

    • 形参为APlayerController* PlayerController和EScreenMoveState ScreenMoveState以及const float &ScreenMoveSpeed = 100.f

  5. 在ScreenMove.cpp中定义MoveDirection函数

    • 声明FVector类型的变量OffsetValue，赋值为FVector::ZeroVector

    • 判断PlayerController是否存在且PlayerController->GetPawn()

    • 通过switch()遍历ScreenMoveState

    • 通过PlayerController调用GetPawn()再调用AddActorWorldOffset()将偏移值OffsetValue传入其中

    • 返回值则直接为return OffsetValue!=FVector::ZeroVector;

  6. 在ScreenMove.h中声明ListenScreenMove函数实现监听屏幕是否移动成功

    • 返回值为bool

    • 形参为APlayerController* PlayerController和const float &ScreenMoveSpeed=100.f

  7. 在ScreenMove.cpp中定义ListenScreenMove函数

    • 直接返回函数MoveDirection

  8. 在TowerDefence.build.cs文件中添加StoneToolClass模块

    • 打开TowerDefence.build.cs，在PublicDependencyModuleNames.AddRange(new string[] { "Core", "CoreUObject", "Engine", "InputCore" });的最后，添加,"StoneToolClass"

  9. 在TowerDefencePlayerController.h文件中添加#include "Tool/ScreenMove.h"

    • 并且在protect下创建一个FScreenMoveUnits结构体的实例ScreenMoveUnits

  10. 在TowerDefencePlayerController.cpp文件中调用

    • 在Tick时间中声明float类型的变量ScreenMoveSpeed并且赋值为100.f

    • 调用ScreenMoveUnits实例中的函数ListenScreenMove()

  11. 在ScreenMove.h中的结构体FScreenMoveUnits前面添加一个导出宏STONETOOLCLASS_API

  12. 在ScreenMove.cpp文件中的MoveDirection()函数中使用switch()遍历ScreenMoveState

    • 每遍历到一个状态值的时候使用OffsetValue=FVector(ScreenMoveSpeed,0.0f,0.0f)来给OffsetValue赋值

    • 在遍历完之后使用PlayerController->GetPawn()->AddActorWorldOffset(OffsetValue);使得玩家控制器偏移

  13. 在TowerDefencePlayerController.cpp中

    • 将ScreenMoveSpeed的值修改为20.0f

    • 在构造函数中设置显示鼠标bShowMouseCursor=true;

  14. 在TowerDefencePlayerController.h中添加函数SetInputModeGameAndUI()

    • 在cpp文件中定义

    • 添加一个FInputModeGameAndUI类型的变量InputMode

    • 设置鼠标一直锁定在视口范围内InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::LockAlways)

    • 关闭按下鼠标则鼠标隐藏功能InputMode.SetHideCursorDuringCapture(false);

    • 将InputMode添加到SetInputMode()函数中作为参数
