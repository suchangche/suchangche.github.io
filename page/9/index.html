UE4-项目-死寂-加载游戏UI界面

发表于 2022-07-15 分类于 unreal engine

加载游戏UI界面

知识点

选择

通过Index引脚选择返回值

随机布尔

字面意思，用于“选择”节点的index引脚为布尔的时候随机输出返回值

使用纹理设置笔刷

可以直接将一个图片当作画刷用，但是通常用于覆盖当前图片

创建控件

以一个类（控件蓝图）来创建新的控件界面

思路

  1. 将玩家点击的地图选择按钮的名称传到“主界面”控件蓝图中

    • 在地图选择按钮控件蓝图中添加命名类型的变量“选择的游戏地图”，勾选其“可编辑实例”以及“生成时公开”

    • 在“地图选择”控件蓝图中选择按钮后设置“选择的游戏地图”

    • 在“主界面”控件蓝图中创建命名类型的变量“选中的地图”

    • 在“地图选择按钮”中获取类的所有控件之后“设置选中的地图”，然后将地图选择按钮中的变量”选择的游戏地图“赋值给它（发生在关闭选中状态之后，开启选中状态之前）

  2. 新建一个控件蓝图“加载界面”

    • 添加一个图像“加载图片组件”作为背景，修改其锚点和偏移

    • 添加一个文本“载入关卡中”，修改其锚点和尺寸，字体，位置

    • 添加一个图像作为加载时闪烁的logo，放在合适的位置

    • 添加一个边界，用于做出游戏黑屏后慢慢显示加载界面的效果

    • 添加一个进度条，设置其位置和尺寸大小，设置其样式中的着色和外观中的”填充颜色和不透明度“

    • 添加动画“淡入”，给边界添加一个轨道，设置其渲染不透明度由1到0

    • 添加动画“闪烁”，给logo添加一个轨道，设置其变换中的缩放从0.1到0.2到0.1；设置其渲染不透明度由0到1到0

    • 添加一个命名类型的变量“选中的地图”，勾选可编辑实例和生成时公开；一个纹理2D类型的变量“加载图片”

    • 在事件构造中判断“选中的地图”是否==“地图名称”

    • 分别通过“随机布尔”和“选择”节点来设置“加载图片”

    • 使用“加载图片组件”调用“使用纹理设置笔刷”来使得“加载图片”覆盖“加载图片组件”

    • 播放“闪烁”动画和“淡入”动画（注意：闪烁动画的循环次数为0：意味着无线循环）

    • 实现进度条更新百分比

      • 首先明确一点，进度条的最大值为1！

      • 创建一个浮点型变量“进度条加载时间”

      • 以事件设置定时器，创建一个自定义事件“Loading”，使用一个分支判断，条件为“进度条加载时间”是否>=1

      • true则”Do Once”，按名称打开关卡，将“选中的地图”作为变量传入

      • false则将“进度条加载时间”+0.01后重新赋值给“进度条加载时间”

  3. 给地图选择界面添加开始游戏按钮及功能

    • 在地图选择控件蓝图中添加“按钮交互”控件蓝图，设置其位置和尺寸以及内容

    • 在主界面拖入地图选择界面，获取“按钮交互”，获取button，绑定事件到点击时，创建自定义事件“游戏加载”

    • 调用“关闭所有UI交互”函数，“创建控件”（类选择为“加载界面”），“添加到视口”，将“选中的地图”变量传入“创建控件”的“选中的地图”引脚中

添加背景音乐

  1. 选择一个背景音乐，点开，设置其“正在循环”为true

  2. 将背景音乐拖入场景中

BUG-不选择地图直接开始游戏

解决思路

在未选择地图的情况下不可以点击开始游戏按钮即可

解决步骤

  1. 在地图选择控件蓝图中选中“开始游戏”按钮交互->行为->已启用->false

  2. 在地图选择按钮控件蓝图中的“开启选中状态”函数中

    • 通过“获取类的所有控件”节点获取“主界面”，从而获取“地图选择界面”，获取“按钮交互”，“设置已启用”为true

    • 关闭选中状态同理设置为false

UE4-项目-死寂-地图选择功能

发表于 2022-07-15 分类于 unreal engine

地图选择功能

知识点

设置笔刷颜色

可以用来设置控件蓝图中组件的颜色

设置渲染范围

用来设置控件蓝图中组件的缩放大小

设置颜色和不透明度

字面意思，控件蓝图中使用

获取类的所有控件

字面意思，可以获取控件蓝图中的组件

思路

  1. 创建一个新的控件蓝图“地图选择按钮”

    • 设置为自定义大小：宽度:550，高度：650

    • 添加一个边界，将其尺寸修改为550，650；渲染不透明度修改为0

    • 添加一个按钮，设置其锚点，尺寸X：530，尺寸Y：630，对其均为0.5；裁切修改为：“剪切到边界-固定（高级）”，添加音效

    • 给按钮下面添加一个图像，对其均为填充，间隙修改为0，渲染变换的缩放变为1.1

    • 设置图像的texture，设置其颜色和不透明度（未选中的时候灰暗一点）

    • 添加一个动画“互动”

      • 给边界添加一个轨道，设置其渲染不透明度从0到1（选中后有边框）

      • 给图像添加一个轨道，设置其颜色和不透明度从0.4到1（选中后变量）

      • 给图像添加一个轨道，设置其变换中的缩放从1.1到1.2（选中后变大）

    • 给按钮添加“悬停时”和“未悬停时”的事件，分别正向和反向播放“互动”动画

  2. 在主界面控件蓝图中，从自定义服装界面中获取按钮交互，再获取其button，”绑定事件到点击时“，自定义事件“地图选择”

  3. 新建一个新的控件蓝图“地图选择”

    • 添加边界和背景模糊，锚点均为填充，添加一个文本“地图选择”，添加返回按钮

    • 添加两个已经制作好的地图选择按钮

  4. 在主界面中添加地图选择界面

    • 修改其锚点为填充，设置其偏移，设置其渲染不透明度为0

    • 添加动画（自定义服装界面和地图选择界面的切换）

      • 复制“点击制作者”动画为“点击地图选择”

      • 将动画中的“制作者界面”替换为“地图选择界面”

      • 将动画中的“主界面”替换为“自定义服装界面”

  5. 切换到地图选择界面之后的摄像机切换

    • 在场景中找一个摄像机作为切换视角的机位

    • 在关卡蓝图中为该摄像机添加一个引用，新建自定义事件“切换到地图选择界面”，使用“获取玩家控制器”和“使用混合设置视图目标”实现切换摄像机功能

  6. 在主界面控件蓝图中调用在关卡蓝图中写好的“切换到地图选择界面”

    • 自定义服装界面中获取“按钮交互”，获取button，绑定事件到点击时，新建自定义事件“地图选择”

    • 在“关闭所有UI交互”函数中添加“地图选择界面”

    • 使用“播放动画”，“执行控制台命令”，“关闭所有UI交互”，“设置可视性”实现自定义服装界面切换到地图选择界面

    • 同理设置地图选择界面返回自定义服装界面

  7. 完善地图选择功能，让玩家选择地图之后，地图缩略图有描边效果，并且保持选中

    • 选中“地图选择按钮”控件蓝图将设计器中的边界组件公开到蓝图中

    • 在图表中新建函数“开启选中状态”，拖入图表中，“设置渲染不透明度”，“设置笔刷颜色”

    • 设置图像，“设置渲染范围”，“设置颜色和不透明度”

    • 同理，制作一个“关闭选中状态”函数

    • 添加bool类型变量“是否被选中”，在“开启选中状态”函数中将其设置为true，在“关闭选中状态”函数中设置为false

    • 在事件图表中，添加一个分支判断“是否被选中”为false的时候才可以播放两个动画

    • 在事件图表中给按钮添加一个点击事件

      • 在点击之后先取消其他被选中的按钮，然后开启当前选中的按钮

      • 调用“获取类的所有控件”节点，class选择“主界面”，获取主界面中的“地图选择界面”，地图选择界面下面的两个“地图选择按钮”，同时调用“关闭选中状态”函数，最后调用“开启选中状态”函数，目标为self

    • 添加不同地图略缩图

      • 在地图选择按钮中添加一个纹理2D类型的变量“场景图片”

      • 勾选“可编辑实例”和“生成时公开”

      • 在图像组件中将“笔刷”和“场景图片”绑定

      • 在”地图选择“控件蓝图中即可替换“地图选择按钮”中的“场景图片”了

优化-选中地图之后返回

选中地图之后返回，第二次进入之后无上次选中的痕迹

思路

  1. 在主界面的地图选择界面返回自定义服装界面逻辑块的最后进行清除上一次的缓存

  2. 拖入“地图选择界面”获取其中的两个地图选择按钮，调用“关闭选中状态”函数

UE4-项目-死寂-本地存储和加载玩家选择数据

发表于 2022-07-14 分类于 unreal engine

本地存储与加载

知识点

SaveGame

一个蓝图类，用于保存游戏数据

游戏存档存在

用于判断游戏的存档是否存在

创建游戏存档对象

字面意思

将游戏保存到插槽

将游戏保存到一个已经创建好的SaveGame中

从插槽加载游戏

字面意思，插槽名称是“游戏存档存在”中插槽的名称

打开关卡（按名称）

字面意思

思路

  1. 将之前选择的服装创建一个存档，方便正式游戏中的角色模型读取其数据来更换模型

    • 创建一个SaveGame类型的蓝图类“BP_ClothSaveGame”

    • 添加骨骼网格体类型的变量“ClothMesh”，为其设置一个默认值

    • 在“BP_MainMenu”中的“切换服装”事件后面判断游戏存档是否存在，不存在则创建一个新的存档

      • 使用“游戏存档存在”判断是否存在（Slot Name设置为Cloth），使用一个分支节点，fasle的时候“创建游戏存档对象”，并将其返回值提升为变量“ClothSaveGame”

      • 拖入ClothSaveGame，引出“set cloth mesh”，将“选择的模型”作为变量传入

      • 调用“将游戏保存到插槽”节点，Object引脚引入“ClothSaveGame”，Slot Name引脚为Cloth

      • 如果有存档，也就是分支节点返回为true，则使用“从插槽加载游戏”，类型转换为“BP_ClothSaveGame”，再设置“ClothSaveGame”，最后设置ClothMesh，然后“将游戏保存到插槽”

  2. 在自定义服装界面添加一个“按钮交互”，文本内容为“完成”，作为选好服装之后进行下一步的操作按钮

  3. 将自定义服装界面选好的服装模型附加到正式游戏中的角色身上

    • 在角色蓝图的事件图表中的“事件开始运行时”开头，判断是或否有存档，有则“从插槽中加载游戏”，类型转换为“BP_ClothSaveGame”，获取其中的ClothMesh“设置骨骼网格体”

    • 注意：无游戏存档的时候，以及转换为”BP_ClothSaveGame“失败时，也需要连接到转换到动画蓝图上去

  4. 测试一下

    • 主界面控件蓝图，获取“自定义服装界面”的“按钮交互”，获取其button，“绑定事件到点击时”，创建自定义事件，“打开关卡（按名称）”

初始化

第二次开启游戏的时候，服装选择界面需要是上一次选择的服装

思路

  1. 在”BP_MainMenu“的事件开始运行时，判断是否有存档，有则“从插槽中加载游戏”，类型转换为“BP_ClothSaveGame”，获取其中的ClothMesh“设置骨骼网格体”

UE4-项目-死寂-自定义服装UI界面

发表于 2022-07-13 更新于 2022-07-14 分类于 unreal engine

自定义服装UI界面

知识点

滚动框

控件蓝图中的组件，一个框里面，物品过多的时候即可滚动物品选择

统一网格面板

空间蓝图中的组件，放在滚动框中

设置骨骼网格体

可以设置骨骼网格体模型，也可以更新模型的时候重置其动画（Reinit Pose为true）

思路

  1. 在世界场景设置中的“游戏模式”，”默认Pawn类“中修改为Pawn

  2. 复制“按钮交互”空间蓝图为“自定义服装按钮”

  3. 删除其中的按钮和文本框，图表中除了播放动画逻辑之外全部删掉，变量“Name”也删掉

  4. 拖入“边界”，将其尺寸xy均修改为150，右上角的“自定义”修改为“所需”，将默认的“渲染不透明度”修改为0

  5. 拖入“按钮”，将其锚点修改为居中，位置均为0，尺寸均为140，对其均为0.5

  6. 添加一个“图像”到“按钮”下面，将水平和竖直对其均修改为填充，将填充的值修改为0

  7. 添加服饰的图像，并将其亮度修改为0.4（在选中的时候才为1）

  8. 将图像的“渲染变换”中的“缩放”均修改为1.1，将按钮的裁切修改为“裁切到边界-固定（高级）”

  9. 将“交互”动画中原本的删掉，添加“图像”的轨道，缩放从1.1到1.2，“颜色和不透明度”的RGBA均从0.4到1

  10. 添加“边界”的轨道，将“渲染不透明度”从0修改为1

  11. 设置按钮的音效

  12. 给按钮添加“悬停时”和“未悬停时”的事件，分别正反向播放“交互”动画

  13. 创建一个“纹理2D”类型的变量“服装图片”，在设计器中的图像组件和“服装图片”进行绑定

  14. 添加“骨骼网格体”类型的变量“服装模型”，勾选其“可编辑实例”和“生成时公开”

  15. 创建新的空间蓝图“自定义服装”

    • 将“游戏声明”控件蓝图中的文本控件和返回按钮复制到“自定义服装中”

    • 添加图像组件，全屏，放置服装选择图片，将一个“滚动框”组件放入图片“服装”栏的下面

    • 在滚动框下面放入一个“统一网格面板”

    • 将“自定义服装按钮”拖入统一网格面板，设置其行列，输入其“服装图片”和“服装模型”

  16. 将“自定义服装”拖入到主界面控件蓝图中

    • 修改其锚点，偏移，渲染不透明度

    • 复制原本的动画，替换为“自定义服装界面”，命名为“点击自定义服装”

    • 将主界面的图表中的“关闭所有UI交互”函数中添加“自定义服装界面”

  17. 在关卡蓝图中添加自定义事件“切换到自定义服装”，使用“获取玩家控制器”，“使用混合设置视图目标”，以及选择一个摄像机的实例实现切换摄像机的效果

  18. 在“主界面”的图表中使用“播放动画”，“控制台命令”，“关闭所有UI交互”，“设置可视性”实现主界面切换到自定义服装界面，同理，也设置自定义界面返回到主界面

  19. 创建一个角色蓝图类“BP_MainMenu”用于选择服装之后展示服装模型

    • 给角色添加一个模型

    • 设置其动画模式为“使用动画资产”，设置一个合适的“要播放的动画”

  20. 将“BP_MainMenu”放入场景中，放在第三个摄像机能看到的位置（自定义服装位置）

  21. 在主界面选择了不同服装之后，角色模型的服装也需要发生相应变化

    • 在“BP_MainMenu”创建一个自定义事件“切换服装”，添加一个骨骼网格体类型的输入“选择的模型”，并提升为变量

    • 使用“设置骨骼网格体”来重置“BP_MainMenu”的模型，并且将“Reinit Pose”为false

    • 在自定义服装按钮控件蓝图中，给button添加一个点击时的事件，通过“获取类的所有actor”调用“BP_MainMenu”中的“切换服装”事件

小结

将 server.cpp 和 client.cpp 分别编译为 server.exe 和 client.exe，先运行 server.exe，再运行 client.exe，输出结果为：
Message form server: Hello World!

Windows 下的 socket 程序和 Linux 思路相同，但细节有所差别：

  1. Windows 下的 socket 程序依赖 Winsock.dll 或 ws2_32.dll，必须提前加载。DLL 有两种加载方式，请查看：动态链接库DLL的加载

  2. Linux 使用“文件描述符”的概念，而 Windows 使用“文件句柄”的概念；Linux 不区分 socket 文件和普通文件，而 Windows 区分；Linux 下 socket() 函数的返回值为 int 类型，而 Windows 下为 SOCKET 类型，也就是句柄。

  3. Linux 下使用 read() / write() 函数读写，而 Windows 下使用 recv() / send() 函数发送和接收。

  4. 关闭 socket 时，Linux 使用 close() 函数，而 Windows 使用 closesocket() 函数。

比特

计算机中数据量单位，也是信息论中信息量的单位，一个比特就是二进制数字的1或者0

8bit=1Byte

固态硬盘厂家和系统的大小不符合是因为厂家的计量单位与系统的不同

速率

连接在计算机网络上的主机在数字信道上传送比特的速率，也成为比特率或数据率

b/s或kb/s

带宽

模拟信号系统中的意义

信号所包含的各自不同频率成分所占据的频率范围

单位：HZ

在计算机网络中的意义

用来表示网络的通信线路所能传送数据的能力，因此网络带宽表示在单位时间内从网络中的某一点到另一点所能通过的“最高数据率”

单位：b/s(kb/s,Mb/s,Gb/s,Tb/s)

一条通信线路的“频带宽度”越宽，其所传输数据的“最高数据率”也越高

吞吐量

  • 表示在单位时间内通过某个网络（或信道，接口）的数据量

  • 吞吐量经常用于对现实世界中网络的一种测量，以便知道实际上到底右多少数据量能通过网络

  • 吞吐量受网络的带宽或额定速率的限制

时延

由四部分构成

  1. 发送时延=分组长度（b）/发送速率（b/s）

  2. 传播时延=信道长度（m）/电磁波传播速率（m/s）

  3. 处理时延（不方便计算）

  4. 排队时延

电磁波传播速率

  1. 自由空间：3*10的8次方m/s

  2. 铜线：2.3*10的8次方m/s

  3. 光纤：2*10的8次方m/s

时延带宽积

传播时延和带宽的乘积

往返时间（RTT）

  • 因特网上的信息不仅是单项传输，而是双向交互

利用率

信道利用率

  • 用来表示信道有百分之几的时间是被利用的（有数据通过）

  • 根据排队论，当某信道的利用率增大时，该信道引起的时延也会迅速增加(车速太快，遇到红绿灯的时候堵车)

  • 所以，信道利用率不是越大越好

网络利用率

全网络的信道利用率的加权平均

丢包率

分组丢失率，在一定时间范围内，传输过程中丢失的分组数量与总分组数量的比率

分组丢失的两种情况

  1. 分组在传输中出现误码，被节点丢弃

  2. 分组到达一台队列已满的分组交换机时被丢弃；在通信量较大时就可能造成网络拥塞

给定一个 n 个元素有序的（升序）整型数组 nums 和一个目标值 target  ，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

示例 2:

提示：

  • 你可以假设 nums 中的所有元素是不重复的。
  • n 将在 [1, 10000]之间。
  • nums 的每个元素都将在 [-9999, 9999]之间。

小结

二分查找的运用条件

  1. 数组必须为有序数组（如果无序，自己排列一下也不是不行）

  2. 数组中元素无重复

    • 因为一旦有重复元素，使用二分查找法返回的元素下标可能不是唯一的

二分查找的逻辑思维

  1. 获取两个下标，一个代表最左边left，一个代表最右边right

  2. 通过两个下表获取一个中间值middle

  3. arr[midlle]和目标值target进行比较，因为数组有序，所以如果middle小于target，则将left更新到middle的位置，反之将right更新到middle

  4. 最后arr[middle]一定会等于target

  5. middle就是值为target的数组元素的下标了！

二分查找的边界条件

也就是区间问题，一般分为两种

  1. 左闭右闭即[left, right]

    • while(left<=right)，使用<=是因为left==right有意义

    • if(nums[middle]>target)时right要赋值为middle-1，因为当前的nums[middle]!=target，所以接下来查找的左区间就为middle-1

  2. 左闭右开即[left, right)

    • while (left < right)，使用<是因为left==right无意义

    • if(nums[middle]>target)时right要赋值为middle，因为当前nums[middle]不等于target，去左区间继续寻找，而寻找区间是左闭右开区间，所以right更新为middle，即：下一个查询区间不会去比较nums[middle]

    • 注意：left依旧要赋值为middle+1，因为左边是闭区间！

思路

  1. 确定是否符合运用条件（有序且无重复数组）

  2. 确定边界条件

  3. 定义left和right进行比较

左闭右闭即[left, right]

左闭右开即[left, right)

搜索插入位置

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

你可以假设数组中无重复元素。

示例 1:

  • 输入: [1,3,5,6], 5
  • 输出: 2

示例 2:

  • 输入: [1,3,5,6], 2
  • 输出: 1

示例 3:

  • 输入: [1,3,5,6], 7
  • 输出: 4

示例 4:

  • 输入: [1,3,5,6], 0
  • 输出: 0

思路

直接上二分查找！

  1. 确定数组有序且无重复元素

  2. 确定解法左闭右闭

  3. 最后想下“返回将会被插入位置的下标”

    • 二分查找最后的查找位置在整个数组中是按序的

    • 要考虑的是最后插入位置和left，middle，right中的哪一个相关

      • 若在[left,right]范围内找不到，那意味着被反复二分查找的地方不存在此数

      • 意味着该数字大于被反复二分查找的那些数

      • 所以应该大于right

    • 最后插入位置应该是right+1

移除元素

给你一个数组 nums 和一个值 val，你需要 原地 移除所有数值等于 val 的元素，并返回移除后数组的新长度。

不要使用额外的数组空间，你必须仅使用 O(1) 额外空间并原地修改输入数组。

元素的顺序可以改变。你不需要考虑数组中超出新长度后面的元素。

双指针法

双指针法（快慢指针法）在数组和链表的操作中是非常常见的，很多考察数组、链表、字符串等操作的面试题，都使用双指针法。

在这里使用快慢指针法！

快慢指针法

通过一个快指针和慢指针在一个for循环下完成两个for循环的工作。

定义快慢指针

  • 快指针：寻找新数组的元素 ，新数组就是不含有目标元素的数组
  • 慢指针：指向更新 新数组下标的位置

  1. 定义快慢两个指针，在一个for循环内

  2. 快指针每循环一次就会更新一次位置，并且每次更新都会将当前指向的值赋值给满指针

  3. 慢指针只有在目标值不等于快指针所指位置的值的时候才会更新

  4. 当快慢指针指向同一个位置的时候自然啥也不会发生，但如果快指针更新一次，慢指针不动，快指针则会将慢指针下一个位置的值赋值给慢指针，达到覆盖了目标值的效果

有序数组的平方

给你一个按 非递减顺序 排序的整数数组 nums，返回 每个数字的平方 组成的新数组，要求也按 非递减顺序 排序。

示例 1：

输入：nums = [-4,-1,0,3,10]

输出：[0,1,9,16,100]

解释：平方后，数组变为 [16,1,0,9,100]，排序后，数组变为 [0,1,9,16,100]

示例 2：

输入：nums = [-7,-3,2,3,11]

输出：[4,9,9,49,121]

双指针法

数组其实是有序的， 只不过负数平方之后可能成为最大数了。

那么数组平方的最大值就在数组的两端，不是最左边就是最右边，不可能是中间。

此时可以考虑双指针法了，i指向起始位置，j指向终止位置。

定义一个新数组result，和A数组一样的大小，让k指向result数组终止位置。

如果A[i] * A[i] < A[j] * A[j] 那么result[k--] = A[j] * A[j]; 。

如果A[i] * A[i] >= A[j] * A[j] 那么result[k--] = A[i] * A[i]; 。

滑动窗口算法

所谓滑动窗口，就是不断的调节子序列的起始位置和终止位置，从而得出我们要想的结果。

在暴力解法中，是一个for循环滑动窗口的起始位置，一个for循环为滑动窗口的终止位置，用两个for循环 完成了一个不断搜索区间的过程。

那么滑动窗口如何用一个for循环来完成这个操作呢。

首先要思考 如果用一个for循环，那么应该表示 滑动窗口的起始位置，还是终止位置。

如果只用一个for循环来表示 滑动窗口的起始位置，那么如何遍历剩下的终止位置？

此时难免再次陷入 暴力解法的怪圈。

所以 只用一个for循环，那么这个循环的索引，一定是表示 滑动窗口的终止位置。

那么问题来了， 滑动窗口的起始位置如何移动呢？

在本题中实现滑动窗口，主要确定如下三点：

  • 窗口内是什么？

窗口就是 满足其和 ≥ s 的长度最小的 连续 子数组。

窗口的起始位置如何移动：如果当前窗口的值大于s了，窗口就要向前移动了（也就是该缩小了）。

窗口的结束位置如何移动：窗口的结束位置就是遍历数组的指针，也就是for循环里的索引。

Socket编程-OSI网络七层模型与TCP/IP协议族

发表于 2022-07-06 更新于 2022-07-07 分类于 Socket

OSI 模型把网络通信的工作分为 7 层，从下到上分别是物理层、数据链路层、网络层、传输层、会话层、表示层和应用层。

OSI 只是存在于概念和理论上的一种模型，它的缺点是分层太多，增加了网络工作的复杂性，所以没有大规模应用。后来人们对 OSI 进行了简化，合并了一些层，最终只保留了 4 层，从下到上分别是接口层、网络层、传输层和应用层，这就是大名鼎鼎的 TCP/IP 模型。

OSI模型与TCP/IP模型

Socket编程位置

 socket 编程，是站在传输层的基础上，所以可以使用 TCP/UDP 协议，但是不能干「访问网页」这样的事情，因为访问网页所需要的 http 协议位于应用层。

两台计算机进行通信时，必须遵守以下原则：

  • 必须是同一层次进行通信，比如，A 计算机的应用层和 B 计算机的传输层就不能通信，因为它们不在一个层次，数据的拆包会遇到问题。
  • 每一层的功能都必须相同，也就是拥有完全相同的网络模型。如果网络模型都不同，那不就乱套了，谁都不认识谁。
  • 数据只能逐层传输，不能跃层。
  • 每一层可以使用下层提供的服务，并向上层提供服务。

TCP/IP协议族

目前实际使用的网络模型是 TCP/IP 模型，它对 OSI 模型进行了简化，只包含了四层，从上到下分别是应用层、传输层、网络层和链路层（网络接口层），每一层都包含了若干协议。

协议（Protocol）就是网络通信过程中的约定或者合同，通信的双方必须都遵守才能正常收发数据。协议有很多种，例如 TCP、UDP、IP 等，通信的双方必须使用同一协议才能通信。协议是一种规范，由计算机组织制定，规定了很多细节，例如，如何建立连接，如何相互识别等。

协议仅仅是一种规范，必须由计算机软件来实现。例如 IP 协议规定了如何找到目标计算机，那么各个开发商在开发自己的软件时就必须遵守该协议，不能另起炉灶。

TCP/IP 模型包含了 TCP、IP、UDP、Telnet、FTP、SMTP 等上百个互为关联的协议，其中 TCP 和 IP 是最常用的两种底层协议，所以把它们统称为“TCP/IP 协议族”。
