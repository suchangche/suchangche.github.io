UE4-项目-死寂-F键拾取物品

F键拾取物品

思路

  1. 创建控件蓝图“拾取提示”

  2. 添加一个水平框，在其下分别添加一个覆层和一个文本“拾取物品”

  3. 设置覆层组件为填充（右边值为15），水平和垂直对其均为填充

  4. 在覆层下面添加一个图片和一个文本”F“，设置两者位置和填充

  5. 在角色蓝图类中的事件图表，事件开始运行时，通过“创建控件”将“拾取提示”创建出来，并且提升为变量“拾取提示”，“添加到视口”，并且”设置可视性“为“已折叠”

  6. 在角色的拾取系统的开启高亮显示的后面设置“拾取提示”的可视性为可视

  7. 在关闭高亮显示的遍历结束之后关闭拾取提示UI的可视性

UE4-项目-死寂-角色动态准星

发表于 2022-07-16 更新于 2022-07-17 分类于 unreal engine

动态准星

知识点

事件Tick

帧更新，它的Delta Seconds引脚则是游戏帧率

已限制映射范围

数学知识里面的映射，可以使得一个小一点的数映射成一个大一点的数，然后原本的数变换，映射之后的数随之变换

In Range A：输入最小值

In Range B：输入最大值

Out Range A：输出最小值

Out Range B：输出最大值

设置渲染平移

可以设置控件蓝图中组件的平移

思路

在角色开枪/跑步的时候准星扩散，在角色不开枪的时候准星减小

  1. 添加变量

    • 在角色蓝图中添加float类型变量“准星扩散当前值”，“准星扩散最大值”，“准星扩散最小值”，“准星扩散速度”

    • “准星扩散最大值”为0.2，准星扩散速度为0.3

  2. 添加函数“准星扩散减小”

    • 给其添加一个float类型的输入“准星扩散减小”

    • “扩散准星当前值”-输入的“准星扩散减小”

    • 判断值是否<=0

      • true则将准星扩散当前值设置为“准星扩散最小值”

      • false则将“扩散准星当前值”-输入的“准星扩散减小”的结果赋值给“准星扩散当前值”

  3. 添加函数“准星扩散增加”

    • 给其添加一个float类型的输入“准星扩散增加”

    • “扩散准星当前值”+输入的“准星扩散增加”

    • 判断值是否>=”准星扩散最大值”

      • true则将准星扩散当前值设置为“准星扩散最大值”

      • false则将“扩散准星当前值”+输入的“准星扩散增加”的结果赋值给“准星扩散当前值”

  4. 调用“准星扩散减小”

    • 在角色蓝图中的事件图表中的“事件Tick”后面的序列节点，添加一个引脚

    • 判断“准星扩散当前值”是否大于等于“准星扩散最小值”，true则调用“准星扩散减小”

    • 拉出事件Tick的Delta Seconds引脚，乘以“准星扩散速度”之后传入“准星扩散减小”函数的引脚

  5. 在“游戏中界面”的设计器中添加动态准星

    • 添加一个画布面板“准星十字”

    • 给其添加一个图片“上”（十字当中的最上面那一笔），将其锚点居中，调整到屏幕合适位置，尺寸修改为条形，添加十字准星的条形纹理图片

    • 同理，创建出“下，左，右”其他三个横条/竖条，合理布局为一个准星的样子

  6. 实现准星动态扩散效果

    • 在游戏中界面的图表中通过“获取类的所有actor”和”BP_Player”通信并将返回值提升为变量“蓝图角色实例”

    • “事件Tick”判断“蓝图角色实例”是否有效，有效则使用序列，分别通过“设置渲染平移”达到使得“上下左右”分别移动的效果

    • 获取”角色蓝图实例“的“准星扩散当前值”，使用“已限制映射范围”将“准星扩散最大值”引入“In Range B”

    • 设置Out Range B为“准星扩散最大值”乘以-200

    • 将计算结果设置给上下左右的“设置渲染平移”

      • 上：直接将结果设置给Y轴平移

      • 下：将结果取绝对值之后设置给Y轴平移

      • 左：同“上”连接到X轴

      • 右：同“下”连接到X轴

  7. 射击时准星持续扩散

    • 在武器父类中通过角色蓝图实例调用“准星扩散增加”

  8. 跑步时准星扩散

    • 在角色蓝图事件Tick后面的序列节点中调用“准星扩散增加”

    • 传入值根据角色速度来计算

      • “获取速度”，“向量长度”，“已限制映射范围”（400，600；0，0.01）

UE4-项目-死寂-角色受伤屏幕效果

发表于 2022-07-16 分类于 unreal engine

角色受伤屏幕变红

思路

  1. 在“游戏中界面”控件蓝图中添加一个图像“受伤屏幕效果”，添加合适的图片，修改为合适的透明度

  2. 修改图像“受伤屏幕效果”的渲染不透明度为0（一开始的时候不可见，被攻击才可见）

  3. 添加动画“受伤”

    • 给“受伤屏幕效果”添加轨道

    • 设置渲染不透明度由0到1

  4. 在图表中添加自定义事件“受伤”

    • 播放动画“受伤”

    • 将Play Mode（播放正反方向）作为形参传出去

  5. 在角色蓝图类中的死亡系统事件图表中的“被击中播放音效且摄像机视角晃动”逻辑段后面

    • 使用一个序列节点，引脚1连接原本的逻辑段

    • 引脚0使用“Do Once”节点后调用“游戏中界面”中的自定义事件“受伤屏幕动画”

    • 添加自定义事件“关闭受伤屏幕效果”，判断玩家是否死亡，未死亡则调用“受伤屏幕动画”，并且反向播放

    • 在序列后面添加一个“以函数名清除定时器”，函数名为“关闭受伤屏幕效果”

    • 紧接着在“以函数名清除定时器”后面添加一个“以函数名设置定时器”，函数名为“关闭受伤屏幕效果”，时间为5s

      • “以函数名设置定时器”作用是没有被攻击之后五秒调用“关闭受伤屏幕效果”

      • “以函数名清除定时器”作用是一旦被攻击，则清除后面的定时器，使得五秒之后不会调用“关闭受伤屏幕效果”

角色生命值绑定

知识点

获取玩家角色

字面意思，通常配合“类型转换为（角色蓝图）”

思路

  1. 给血量条创建绑定

    • 点击血量条，百分比，创建绑定

    • 获取玩家角色，类型转换为BP_Player

    • 获取“当前血量”，“（浮点）/（浮点）”，连接到返回值

      • 因为百分比最大是1，角色血量最大是100，要达到能和百分比搭配的地步

  2. 绑定血量条颜色（底血量的情况下颜色变红）

    • 进度条->外观->填充颜色和不透明度->绑定->创建绑定

    • 大体逻辑和血量条数值绑定一样，只有最后的输出不一样

    • 返回节点创建颜色，当前血量/100之后连接到G,B引脚上,R引脚设置为1

      • 颜色是由R(红色),G(绿色),B(蓝色),A(透明度)组成的，取值范围是[0,1]，当全为1的时候则为白色，当一个值为1，其他值为0时，则显示为1的那个颜色

子弹数量绑定

思路

  1. 点击“子弹剩余数量”这个文本->内容->文本->绑定->创建绑定

    • 获取玩家角色，类型转换为BP_Player

    • 判断是否装备主武器

      • true则获取主武器实例，获取子弹数量，输出

      • false则判断是否装备副武器

    • 是否装备副武器

      • true则获取副武器实例，获取子弹数量，输出

      • false则输出0

  2. 绑定弹匣数量

    • 选中“弹匣数量”文本，创建绑定

    • 思路和绑定子弹数量一样，只不过将子弹数量换成弹匣数量

  3. 将角色UI添加到视口

    • 因为是有可操纵角色的时候才能有这个UI，所以打开“BP_Player”

    • 在事件图表中的“事件开始运行时”最后面“创建控件”，class选择“游戏中界面”，将返回值提升为变量“游戏中界面”，将其“添加到视口”

UE4-项目-死寂-加载游戏UI界面

发表于 2022-07-15 分类于 unreal engine

加载游戏UI界面

知识点

选择

通过Index引脚选择返回值

随机布尔

字面意思，用于“选择”节点的index引脚为布尔的时候随机输出返回值

使用纹理设置笔刷

可以直接将一个图片当作画刷用，但是通常用于覆盖当前图片

创建控件

以一个类（控件蓝图）来创建新的控件界面

思路

  1. 将玩家点击的地图选择按钮的名称传到“主界面”控件蓝图中

    • 在地图选择按钮控件蓝图中添加命名类型的变量“选择的游戏地图”，勾选其“可编辑实例”以及“生成时公开”

    • 在“地图选择”控件蓝图中选择按钮后设置“选择的游戏地图”

    • 在“主界面”控件蓝图中创建命名类型的变量“选中的地图”

    • 在“地图选择按钮”中获取类的所有控件之后“设置选中的地图”，然后将地图选择按钮中的变量”选择的游戏地图“赋值给它（发生在关闭选中状态之后，开启选中状态之前）

  2. 新建一个控件蓝图“加载界面”

    • 添加一个图像“加载图片组件”作为背景，修改其锚点和偏移

    • 添加一个文本“载入关卡中”，修改其锚点和尺寸，字体，位置

    • 添加一个图像作为加载时闪烁的logo，放在合适的位置

    • 添加一个边界，用于做出游戏黑屏后慢慢显示加载界面的效果

    • 添加一个进度条，设置其位置和尺寸大小，设置其样式中的着色和外观中的”填充颜色和不透明度“

    • 添加动画“淡入”，给边界添加一个轨道，设置其渲染不透明度由1到0

    • 添加动画“闪烁”，给logo添加一个轨道，设置其变换中的缩放从0.1到0.2到0.1；设置其渲染不透明度由0到1到0

    • 添加一个命名类型的变量“选中的地图”，勾选可编辑实例和生成时公开；一个纹理2D类型的变量“加载图片”

    • 在事件构造中判断“选中的地图”是否==“地图名称”

    • 分别通过“随机布尔”和“选择”节点来设置“加载图片”

    • 使用“加载图片组件”调用“使用纹理设置笔刷”来使得“加载图片”覆盖“加载图片组件”

    • 播放“闪烁”动画和“淡入”动画（注意：闪烁动画的循环次数为0：意味着无线循环）

    • 实现进度条更新百分比

      • 首先明确一点，进度条的最大值为1！

      • 创建一个浮点型变量“进度条加载时间”

小结

将 server.cpp 和 client.cpp 分别编译为 server.exe 和 client.exe，先运行 server.exe，再运行 client.exe，输出结果为：
Message form server: Hello World!

Windows 下的 socket 程序和 Linux 思路相同，但细节有所差别：

  1. Windows 下的 socket 程序依赖 Winsock.dll 或 ws2_32.dll，必须提前加载。DLL 有两种加载方式，请查看：动态链接库DLL的加载

  2. Linux 使用“文件描述符”的概念，而 Windows 使用“文件句柄”的概念；Linux 不区分 socket 文件和普通文件，而 Windows 区分；Linux 下 socket() 函数的返回值为 int 类型，而 Windows 下为 SOCKET 类型，也就是句柄。

  3. Linux 下使用 read() / write() 函数读写，而 Windows 下使用 recv() / send() 函数发送和接收。

  4. 关闭 socket 时，Linux 使用 close() 函数，而 Windows 使用 closesocket() 函数。
