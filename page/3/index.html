UE4-项目-死寂-角色受伤屏幕效果

发表于 2022-07-16 分类于 unreal engine

角色受伤屏幕变红

思路

  1. 在“游戏中界面”控件蓝图中添加一个图像“受伤屏幕效果”，添加合适的图片，修改为合适的透明度

  2. 修改图像“受伤屏幕效果”的渲染不透明度为0（一开始的时候不可见，被攻击才可见）

  3. 添加动画“受伤”

    • 给“受伤屏幕效果”添加轨道

    • 设置渲染不透明度由0到1

  4. 在图表中添加自定义事件“受伤”

    • 播放动画“受伤”

    • 将Play Mode（播放正反方向）作为形参传出去

  5. 在角色蓝图类中的死亡系统事件图表中的“被击中播放音效且摄像机视角晃动”逻辑段后面

    • 使用一个序列节点，引脚1连接原本的逻辑段

    • 引脚0使用“Do Once”节点后调用“游戏中界面”中的自定义事件“受伤屏幕动画”

    • 添加自定义事件“关闭受伤屏幕效果”，判断玩家是否死亡，未死亡则调用“受伤屏幕动画”，并且反向播放

    • 在序列后面添加一个“以函数名清除定时器”，函数名为“关闭受伤屏幕效果”

    • 紧接着在“以函数名清除定时器”后面添加一个“以函数名设置定时器”，函数名为“关闭受伤屏幕效果”，时间为5s

      • “以函数名设置定时器”作用是没有被攻击之后五秒调用“关闭受伤屏幕效果”

      • “以函数名清除定时器”作用是一旦被攻击，则清除后面的定时器，使得五秒之后不会调用“关闭受伤屏幕效果”

UE4-项目-死寂-角色UI

发表于 2022-07-16 分类于 unreal engine

角色属性UI排版

思路

  1. 创建新的控件蓝图“游戏中界面”

    • 添加一个文本作为游戏声明，放在不起眼的位置，颜色调整为灰色

    • 添加一个新的画布面板“子弹和生命值”

      • 添加一个图像放在右下角作为主武器还是副武器的背景图

      • 添加一个文本放在图像上面用于显示当前弹匣数量

      • 添加一个文本显示子弹数量放在图像的左边

      • 添加一个图像，拉长成为细线，作为和下面血条的分割线

      • 添加一个进度条作为血量条，设置其背景图，填充图

角色生命值绑定

知识点

获取玩家角色

字面意思，通常配合“类型转换为（角色蓝图）”

思路

  1. 给血量条创建绑定

    • 点击血量条，百分比，创建绑定

    • 获取玩家角色，类型转换为BP_Player

    • 获取“当前血量”，“（浮点）/（浮点）”，连接到返回值

      • 因为百分比最大是1，角色血量最大是100，要达到能和百分比搭配的地步

  2. 绑定血量条颜色（底血量的情况下颜色变红）

    • 进度条->外观->填充颜色和不透明度->绑定->创建绑定

    • 大体逻辑和血量条数值绑定一样，只有最后的输出不一样

    • 返回节点创建颜色，当前血量/100之后连接到G,B引脚上,R引脚设置为1

      • 颜色是由R(红色),G(绿色),B(蓝色),A(透明度)组成的，取值范围是[0,1]，当全为1的时候则为白色，当一个值为1，其他值为0时，则显示为1的那个颜色

子弹数量绑定

思路

  1. 点击“子弹剩余数量”这个文本->内容->文本->绑定->创建绑定

    • 获取玩家角色，类型转换为BP_Player

    • 判断是否装备主武器

      • true则获取主武器实例，获取子弹数量，输出

      • false则判断是否装备副武器

    • 是否装备副武器

      • true则获取副武器实例，获取子弹数量，输出

      • false则输出0

  2. 绑定弹匣数量

    • 选中“弹匣数量”文本，创建绑定

    • 思路和绑定子弹数量一样，只不过将子弹数量换成弹匣数量

  3. 将角色UI添加到视口

    • 因为是有可操纵角色的时候才能有这个UI，所以打开“BP_Player”

    • 在事件图表中的“事件开始运行时”最后面“创建控件”，class选择“游戏中界面”，将返回值提升为变量“游戏中界面”，将其“添加到视口”

UE4-项目-死寂-加载游戏UI界面

发表于 2022-07-15 分类于 unreal engine

加载游戏UI界面

知识点

选择

通过Index引脚选择返回值

随机布尔

字面意思，用于“选择”节点的index引脚为布尔的时候随机输出返回值

使用纹理设置笔刷

可以直接将一个图片当作画刷用，但是通常用于覆盖当前图片

创建控件

以一个类（控件蓝图）来创建新的控件界面

思路

  1. 将玩家点击的地图选择按钮的名称传到“主界面”控件蓝图中

    • 在地图选择按钮控件蓝图中添加命名类型的变量“选择的游戏地图”，勾选其“可编辑实例”以及“生成时公开”

    • 在“地图选择”控件蓝图中选择按钮后设置“选择的游戏地图”

    • 在“主界面”控件蓝图中创建命名类型的变量“选中的地图”

    • 在“地图选择按钮”中获取类的所有控件之后“设置选中的地图”，然后将地图选择按钮中的变量”选择的游戏地图“赋值给它（发生在关闭选中状态之后，开启选中状态之前）

  2. 新建一个控件蓝图“加载界面”

    • 添加一个图像“加载图片组件”作为背景，修改其锚点和偏移

    • 添加一个文本“载入关卡中”，修改其锚点和尺寸，字体，位置

    • 添加一个图像作为加载时闪烁的logo，放在合适的位置

    • 添加一个边界，用于做出游戏黑屏后慢慢显示加载界面的效果

    • 添加一个进度条，设置其位置和尺寸大小，设置其样式中的着色和外观中的”填充颜色和不透明度“

    • 添加动画“淡入”，给边界添加一个轨道，设置其渲染不透明度由1到0

    • 添加动画“闪烁”，给logo添加一个轨道，设置其变换中的缩放从0.1到0.2到0.1；设置其渲染不透明度由0到1到0

    • 添加一个命名类型的变量“选中的地图”，勾选可编辑实例和生成时公开；一个纹理2D类型的变量“加载图片”

    • 在事件构造中判断“选中的地图”是否==“地图名称”

    • 分别通过“随机布尔”和“选择”节点来设置“加载图片”

    • 使用“加载图片组件”调用“使用纹理设置笔刷”来使得“加载图片”覆盖“加载图片组件”

    • 播放“闪烁”动画和“淡入”动画（注意：闪烁动画的循环次数为0：意味着无线循环）

    • 实现进度条更新百分比

      • 首先明确一点，进度条的最大值为1！

      • 创建一个浮点型变量“进度条加载时间”

      • 以事件设置定时器，创建一个自定义事件“Loading”，使用一个分支判断，条件为“进度条加载时间”是否>=1

      • true则”Do Once”，按名称打开关卡，将“选中的地图”作为变量传入

      • false则将“进度条加载时间”+0.01后重新赋值给“进度条加载时间”

  3. 给地图选择界面添加开始游戏按钮及功能

    • 在地图选择控件蓝图中添加“按钮交互”控件蓝图，设置其位置和尺寸以及内容

    • 在主界面拖入地图选择界面，获取“按钮交互”，获取button，绑定事件到点击时，创建自定义事件“游戏加载”

    • 调用“关闭所有UI交互”函数，“创建控件”（类选择为“加载界面”），“添加到视口”，将“选中的地图”变量传入“创建控件”的“选中的地图”引脚中

添加背景音乐

  1. 选择一个背景音乐，点开，设置其“正在循环”为true

  2. 将背景音乐拖入场景中

BUG-不选择地图直接开始游戏

解决思路

在未选择地图的情况下不可以点击开始游戏按钮即可

解决步骤

  1. 在地图选择控件蓝图中选中“开始游戏”按钮交互->行为->已启用->false

  2. 在地图选择按钮控件蓝图中的“开启选中状态”函数中

    • 通过“获取类的所有控件”节点获取“主界面”，从而获取“地图选择界面”，获取“按钮交互”，“设置已启用”为true

    • 关闭选中状态同理设置为false

UE4-项目-死寂-地图选择功能

发表于 2022-07-15 分类于 unreal engine

地图选择功能

知识点

设置笔刷颜色

可以用来设置控件蓝图中组件的颜色

设置渲染范围

用来设置控件蓝图中组件的缩放大小

设置颜色和不透明度

字面意思，控件蓝图中使用

获取类的所有控件

字面意思，可以获取控件蓝图中的组件

思路

  1. 创建一个新的控件蓝图“地图选择按钮”

    • 设置为自定义大小：宽度:550，高度：650

    • 添加一个边界，将其尺寸修改为550，650；渲染不透明度修改为0

    • 添加一个按钮，设置其锚点，尺寸X：530，尺寸Y：630，对其均为0.5；裁切修改为：“剪切到边界-固定（高级）”，添加音效

    • 给按钮下面添加一个图像，对其均为填充，间隙修改为0，渲染变换的缩放变为1.1

    • 设置图像的texture，设置其颜色和不透明度（未选中的时候灰暗一点）

    • 添加一个动画“互动”

      • 给边界添加一个轨道，设置其渲染不透明度从0到1（选中后有边框）

      • 给图像添加一个轨道，设置其颜色和不透明度从0.4到1（选中后变量）

      • 给图像添加一个轨道，设置其变换中的缩放从1.1到1.2（选中后变大）

    • 给按钮添加“悬停时”和“未悬停时”的事件，分别正向和反向播放“互动”动画

  2. 在主界面控件蓝图中，从自定义服装界面中获取按钮交互，再获取其button，”绑定事件到点击时“，自定义事件“地图选择”

  3. 新建一个新的控件蓝图“地图选择”

    • 添加边界和背景模糊，锚点均为填充，添加一个文本“地图选择”，添加返回按钮

    • 添加两个已经制作好的地图选择按钮

  4. 在主界面中添加地图选择界面

    • 修改其锚点为填充，设置其偏移，设置其渲染不透明度为0

    • 添加动画（自定义服装界面和地图选择界面的切换）

      • 复制“点击制作者”动画为“点击地图选择”

      • 将动画中的“制作者界面”替换为“地图选择界面”

      • 将动画中的“主界面”替换为“自定义服装界面”

  5. 切换到地图选择界面之后的摄像机切换

    • 在场景中找一个摄像机作为切换视角的机位

    • 在关卡蓝图中为该摄像机添加一个引用，新建自定义事件“切换到地图选择界面”，使用“获取玩家控制器”和“使用混合设置视图目标”实现切换摄像机功能

  6. 在主界面控件蓝图中调用在关卡蓝图中写好的“切换到地图选择界面”

    • 自定义服装界面中获取“按钮交互”，获取button，绑定事件到点击时，新建自定义事件“地图选择”

    • 在“关闭所有UI交互”函数中添加“地图选择界面”

    • 使用“播放动画”，“执行控制台命令”，“关闭所有UI交互”，“设置可视性”实现自定义服装界面切换到地图选择界面

    • 同理设置地图选择界面返回自定义服装界面

  7. 完善地图选择功能，让玩家选择地图之后，地图缩略图有描边效果，并且保持选中

    • 选中“地图选择按钮”控件蓝图将设计器中的边界组件公开到蓝图中

    • 在图表中新建函数“开启选中状态”，拖入图表中，“设置渲染不透明度”，“设置笔刷颜色”

    • 设置图像，“设置渲染范围”，“设置颜色和不透明度”

    • 同理，制作一个“关闭选中状态”函数

    • 添加bool类型变量“是否被选中”，在“开启选中状态”函数中将其设置为true，在“关闭选中状态”函数中设置为false

    • 在事件图表中，添加一个分支判断“是否被选中”为false的时候才可以播放两个动画

    • 在事件图表中给按钮添加一个点击事件

      • 在点击之后先取消其他被选中的按钮，然后开启当前选中的按钮

      • 调用“获取类的所有控件”节点，class选择“主界面”，获取主界面中的“地图选择界面”，地图选择界面下面的两个“地图选择按钮”，同时调用“关闭选中状态”函数，最后调用“开启选中状态”函数，目标为self

    • 添加不同地图略缩图

      • 在地图选择按钮中添加一个纹理2D类型的变量“场景图片”

      • 勾选“可编辑实例”和“生成时公开”

      • 在图像组件中将“笔刷”和“场景图片”绑定

      • 在”地图选择“控件蓝图中即可替换“地图选择按钮”中的“场景图片”了

优化-选中地图之后返回

选中地图之后返回，第二次进入之后无上次选中的痕迹

思路

  1. 在主界面的地图选择界面返回自定义服装界面逻辑块的最后进行清除上一次的缓存

  2. 拖入“地图选择界面”获取其中的两个地图选择按钮，调用“关闭选中状态”函数

UE4-项目-死寂-本地存储和加载玩家选择数据

发表于 2022-07-14 分类于 unreal engine

本地存储与加载

知识点

SaveGame

一个蓝图类，用于保存游戏数据

游戏存档存在

用于判断游戏的存档是否存在

创建游戏存档对象

字面意思

将游戏保存到插槽

将游戏保存到一个已经创建好的SaveGame中

从插槽加载游戏

字面意思，插槽名称是“游戏存档存在”中插槽的名称

打开关卡（按名称）

字面意思

思路

  1. 将之前选择的服装创建一个存档，方便正式游戏中的角色模型读取其数据来更换模型

    • 创建一个SaveGame类型的蓝图类“BP_ClothSaveGame”

    • 添加骨骼网格体类型的变量“ClothMesh”，为其设置一个默认值

    • 在“BP_MainMenu”中的“切换服装”事件后面判断游戏存档是否存在，不存在则创建一个新的存档

      • 使用“游戏存档存在”判断是否存在（Slot Name设置为Cloth），使用一个分支节点，fasle的时候“创建游戏存档对象”，并将其返回值提升为变量“ClothSaveGame”

      • 拖入ClothSaveGame，引出“set cloth mesh”，将“选择的模型”作为变量传入

      • 调用“将游戏保存到插槽”节点，Object引脚引入“ClothSaveGame”，Slot Name引脚为Cloth

      • 如果有存档，也就是分支节点返回为true，则使用“从插槽加载游戏”，类型转换为“BP_ClothSaveGame”，再设置“ClothSaveGame”，最后设置ClothMesh，然后“将游戏保存到插槽”

  2. 在自定义服装界面添加一个“按钮交互”，文本内容为“完成”，作为选好服装之后进行下一步的操作按钮

  3. 将自定义服装界面选好的服装模型附加到正式游戏中的角色身上

    • 在角色蓝图的事件图表中的“事件开始运行时”开头，判断是或否有存档，有则“从插槽中加载游戏”，类型转换为“BP_ClothSaveGame”，获取其中的ClothMesh“设置骨骼网格体”

    • 注意：无游戏存档的时候，以及转换为”BP_ClothSaveGame“失败时，也需要连接到转换到动画蓝图上去

  4. 测试一下

    • 主界面控件蓝图，获取“自定义服装界面”的“按钮交互”，获取其button，“绑定事件到点击时”，创建自定义事件，“打开关卡（按名称）”

初始化

第二次开启游戏的时候，服装选择界面需要是上一次选择的服装

思路

  1. 在”BP_MainMenu“的事件开始运行时，判断是否有存档，有则“从插槽中加载游戏”，类型转换为“BP_ClothSaveGame”，获取其中的ClothMesh“设置骨骼网格体”

UE4-项目-死寂-自定义服装UI界面

发表于 2022-07-13 更新于 2022-07-14 分类于 unreal engine

自定义服装UI界面

知识点

滚动框

控件蓝图中的组件，一个框里面，物品过多的时候即可滚动物品选择

统一网格面板

空间蓝图中的组件，放在滚动框中

设置骨骼网格体

可以设置骨骼网格体模型，也可以更新模型的时候重置其动画（Reinit Pose为true）

思路

  1. 在世界场景设置中的“游戏模式”，”默认Pawn类“中修改为Pawn

  2. 复制“按钮交互”空间蓝图为“自定义服装按钮”

  3. 删除其中的按钮和文本框，图表中除了播放动画逻辑之外全部删掉，变量“Name”也删掉

  4. 拖入“边界”，将其尺寸xy均修改为150，右上角的“自定义”修改为“所需”，将默认的“渲染不透明度”修改为0

  5. 拖入“按钮”，将其锚点修改为居中，位置均为0，尺寸均为140，对其均为0.5

  6. 添加一个“图像”到“按钮”下面，将水平和竖直对其均修改为填充，将填充的值修改为0

  7. 添加服饰的图像，并将其亮度修改为0.4（在选中的时候才为1）

  8. 将图像的“渲染变换”中的“缩放”均修改为1.1，将按钮的裁切修改为“裁切到边界-固定（高级）”

Windows 下的 socket 程序和 Linux 思路相同，但细节有所差别：

  1. Windows 下的 socket 程序依赖 Winsock.dll 或 ws2_32.dll，必须提前加载。DLL 有两种加载方式，请查看：动态链接库DLL的加载

  2. Linux 使用“文件描述符”的概念，而 Windows 使用“文件句柄”的概念；Linux 不区分 socket 文件和普通文件，而 Windows 区分；Linux 下 socket() 函数的返回值为 int 类型，而 Windows 下为 SOCKET 类型，也就是句柄。

  3. Linux 下使用 read() / write() 函数读写，而 Windows 下使用 recv() / send() 函数发送和接收。

  4. 关闭 socket 时，Linux 使用 close() 函数，而 Windows 使用 closesocket() 函数。

计算机网络-性能指标

发表于 2022-07-07 分类于 计算机网络

常用性能指标

  1. 速率

  2. 带宽

  3. 吞吐量

  4. 时延

  5. 时延带宽积

  6. 往返时间

  7. 利用率

  8. 丢包率

比特

计算机中数据量单位，也是信息论中信息量的单位，一个比特就是二进制数字的1或者0

8bit=1Byte

固态硬盘厂家和系统的大小不符合是因为厂家的计量单位与系统的不同

速率

连接在计算机网络上的主机在数字信道上传送比特的速率，也成为比特率或数据率

b/s或kb/s

带宽

模拟信号系统中的意义

信号所包含的各自不同频率成分所占据的频率范围

单位：HZ

在计算机网络中的意义

用来表示网络的通信线路所能传送数据的能力，因此网络带宽表示在单位时间内从网络中的某一点到另一点所能通过的“最高数据率”

单位：b/s(kb/s,Mb/s,Gb/s,Tb/s)

一条通信线路的“频带宽度”越宽，其所传输数据的“最高数据率”也越高

吞吐量

  • 表示在单位时间内通过某个网络（或信道，接口）的数据量

  • 吞吐量经常用于对现实世界中网络的一种测量，以便知道实际上到底右多少数据量能通过网络

  • 吞吐量受网络的带宽或额定速率的限制

时延

由四部分构成

  1. 发送时延=分组长度（b）/发送速率（b/s）

  2. 传播时延=信道长度（m）/电磁波传播速率（m/s）

  3. 处理时延（不方便计算）

  4. 排队时延

电磁波传播速率

  1. 自由空间：3*10的8次方m/s

  2. 铜线：2.3*10的8次方m/s

  3. 光纤：2*10的8次方m/s

时延带宽积

传播时延和带宽的乘积

往返时间（RTT）

  • 因特网上的信息不仅是单项传输，而是双向交互

利用率

信道利用率

  • 用来表示信道有百分之几的时间是被利用的（有数据通过）

  • 根据排队论，当某信道的利用率增大时，该信道引起的时延也会迅速增加(车速太快，遇到红绿灯的时候堵车)

  • 所以，信道利用率不是越大越好

网络利用率

全网络的信道利用率的加权平均

丢包率

分组丢失率，在一定时间范围内，传输过程中丢失的分组数量与总分组数量的比率

分组丢失的两种情况

  1. 分组在传输中出现误码，被节点丢弃

  2. 分组到达一台队列已满的分组交换机时被丢弃；在通信量较大时就可能造成网络拥塞
