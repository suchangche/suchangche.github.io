知识点

GetDefault

GetDefault<类>()

获取类的默认对象。在大多数情况下，不应修改类默认对象。因此，此方法返回一个不可变指针。如果需要修改默认对象，请改用 GetMutableDefault。

FindByPredicate(）

是TArray的一种高级查找方式，通常在()内使用Lambda表达式/匿名函数实现复杂的查找

Lambda表达式/匿名函数

  1. [外部变量方位方式说明符]
    [ ] 方括号用于向编译器表明当前是一个 lambda 表达式，其不能被省略。在方括号内部，可以注明当前 lambda 函数的函数体中可以使用哪些“外部变量”。

所谓外部变量，指的是和当前 lambda 表达式位于同一作用域内的所有局部变量。

  1. (参数)
    和普通函数的定义一样，lambda 匿名函数也可以接收外部传递的多个参数。和普通函数不同的是，如果不需要传递参数，可以连同 () 小括号一起省略；

3) mutable
此关键字可以省略，如果使用则之前的 () 小括号将不能省略（参数个数可以为 0）。默认情况下，对于以值传递方式引入的外部变量，不允许在 lambda 表达式内部修改它们的值（可以理解为这部分变量都是 const 常量）。而如果想修改它们，就必须使用 mutable 关键字。

注意，对于以值传递方式引入的外部变量，lambda 表达式修改的是拷贝的那一份，并不会修改真正的外部变量；

  1. noexcept/throw()
    可以省略，如果使用，在之前的 () 小括号将不能省略（参数个数可以为 0）。默认情况下，lambda 函数的函数体中可以抛出任何类型的异常。而标注 noexcept 关键字，则表示函数体内不会抛出任何异常；使用 throw() 可以指定 lambda 函数内部可以抛出的异常类型。

值得一提的是，如果 lambda 函数标有 noexcept 而函数体内抛出了异常，又或者使用 throw() 限定了异常类型而函数体内抛出了非指定类型的异常，这些异常无法使用 try-catch 捕获，会导致程序执行失败（本节后续会给出实例）。

  1. -> 返回值类型
    指明 lambda 匿名函数的返回值类型。值得一提的是，如果 lambda 函数体内只有一个 return 语句，或者该函数返回 void，则编译器可以自行推断出返回值类型，此情况下可以直接省略-> 返回值类型。

  2. 函数体
    和普通函数一样，lambda 匿名函数包含的内部代码都放置在函数体中。该函数体内除了可以使用指定传递进来的参数之外，还可以使用指定的外部变量以及全局范围内的所有全局变量。

外部变量格式 功能
[] 空方括号表示当前 lambda 匿名函数中不导入任何外部变量。
[=] 只有一个 = 等号，表示以值传递的方式导入所有外部变量；
[&] 只有一个 & 符号，表示以引用传递的方式导入所有外部变量；
[val1,val2,…] 表示以值传递的方式导入 val1、val2 等指定的外部变量，同时多个变量之间没有先后次序；
[&val1,&val2,…] 表示以引用传递的方式导入 val1、val2等指定的外部变量，多个变量之间没有前后次序；
[val,&val2,…] 以上 2 种方式还可以混合使用，变量之间没有前后次序。
[=,&val1,…] 表示除 val1 以引用传递的方式导入外，其它外部变量都以值传递的方式导入。
[this] 表示以值传递的方式导入当前的 this 指针。

 注意，单个外部变量不允许以相同的传递方式导入多次。例如 [=，val1] 中，val1 先后被以值传递的方式导入了 2 次，这是非法的。

知识点

ConstructorHelpers::FClassFinder<>()

用于获取某个蓝图的类型class

在使用它们的时候还得遵守如下规则：

  • 只能在类的构造函数中使用，如果在普通的逻辑代码中嵌套这份代码，会引起整个编译器的crash。（实际上里面代码就有检查是否在构造函数里，否则crash）

  • 其次，FObjectFinder/FClassFinder变量必须是static的，从而保证只有一份资源实例。

  1. **FObjectFinder**：一般用来加载非蓝图资源，比如StaticMesh、Material、SoundWave、ParticlesSystem、AnimSequence、SkeletalMesh等资源：
  1. **FClassFinder**：一般用来加载蓝图资源并获取蓝图Class。这是因为如果C++要用蓝图创建对象，必须先获取蓝图的Class，然后再通过Class生成蓝图对象：
  • FClassFinder的模版名不能直接写UBlueprint，例如：FClassFinder是错误的。创建蓝图时选择的是什么父类，则写对应的父类名，假如是Actor，那么要写成：FClassFinder，否则无法加载成功。
  • FClassFinder的模版名必须和TSubclassOf变量的模版名一致，当然也可使用UClass代替TSubclassOf。实际上TSubclassOf也是UClass，只是更加强调这个Class是从T派生出来的。
  • 在启动游戏时若报错提示找不到文件而崩溃（例如：Default property warnings and errors:Error: COD Constructor (MyGameMode): Failed to find /Game/MyProject/MyBP.MyBP）
    这是因为UE4资源路径的一个规范问题，解决办法有两种：
    1. 在copy reference出来的文件路径后面加_C，例如：”Blueprint’/Game/Blueprints/MyBP.MyBP_C’”（_C可以理解为获取Class的意思）。
    2. 去掉路径前缀，例如：”/Game/Blueprints/MyBP”

封装屏幕移动功能到插件

效果：鼠标移动到屏幕边缘就可以造成屏幕相应方向的移动

知识点

UE的屏幕坐标

UE4的屏幕默认左上角为(0,0)

IsNearlyEqual()

源码

bool IsNearlyEqual(float A, float B, float ErrorTolerance = SMALL_NUMBER)

含义

A-B，如果值在ErrorTolerance之间，则返回true

注意事项

处于FMath命名空间中，在之前需要先FMath::

例如：

PlayerController->GetViewportSize()

源码

含义

获取屏幕大小，并将其X，Y值传给SizeX和SizeX

注意

必须由PlayerController->调用

PlayerController->GetMousePosition()

源码

含义

获取鼠标位置并将其传给参数LocationX和LocationY

注意

依旧需要PlayerController->调用

bShowMouseCursor

含义

为true则显示鼠标；为false则不显示鼠标

AddActorWorldOffset()

源码

含义

根据输入的偏移值（FVector类型）来使得物体进行位置偏移

FInputModeGameAndUI

一种数据结构，可以设置游戏和UI的输入模式

SetInputMode()

设置输入模式，其参数可以为FInputModeGameAndUI类型的变量

思路

  1. 在屏幕设置一个长方形的边界，在此长方形之内，鼠标随意移动，一旦超出边界，则屏幕往相应位置移动

  1. 打开TowerDefence\Plugins\StoneToolClass\Source\StoneToolClass\Public\Tool\文件夹，创建ScreenMove.h,并且在相应的Private\Tool\文件夹下面创建ScreenMove.cpp文件

    • 创建一个结构体FScreenMoveUnits,在其中声明一个以EScreenMoveUnits为返回值的函数CursorMove，形参是APlayerController的实例，并且在.CPP文件中创建该函数的定义

  3. 在ScreenMove.cpp中定义CursorMove函数

    • 判断PlayerController是否有效，如果为true，则

    • 定义屏幕尺寸SizeX和SizeY，初始值赋值为INDEX_NONE

    • 通过APlayerController->GetViewportSize()获取屏幕尺寸并且传参给SizeX和SizeY

    • 通过PlayerController->GetMousePosition()获取鼠标位置并且传参给MousePostionX，MousePostionY

    • 用if()来限定MousePostionX，MousePostionX的取值范围（<=SizeX和SizeY&&>=0）

    • 通过FMath::IsNearlyEqual()来修改EScreenMoveUnits的状态，分别修改为上下左右，左上左下，右上右下；如果以上状态均不是，则返回为Screen_None

UE4-项目-死寂-角色动态准星

发表于 2022-07-16 更新于 2022-07-17 分类于 unreal engine

动态准星

知识点

事件Tick

帧更新，它的Delta Seconds引脚则是游戏帧率

已限制映射范围

数学知识里面的映射，可以使得一个小一点的数映射成一个大一点的数，然后原本的数变换，映射之后的数随之变换

In Range A：输入最小值

In Range B：输入最大值

Out Range A：输出最小值

Out Range B：输出最大值

设置渲染平移

可以设置控件蓝图中组件的平移

思路

在角色开枪/跑步的时候准星扩散，在角色不开枪的时候准星减小

  1. 添加变量

    • 在角色蓝图中添加float类型变量“准星扩散当前值”，“准星扩散最大值”，“准星扩散最小值”，“准星扩散速度”

    • “准星扩散最大值”为0.2，准星扩散速度为0.3

  2. 添加函数“准星扩散减小”

    • 给其添加一个float类型的输入“准星扩散减小”

    • “扩散准星当前值”-输入的“准星扩散减小”

    • 判断值是否<=0

      • true则将准星扩散当前值设置为“准星扩散最小值”

      • false则将“扩散准星当前值”-输入的“准星扩散减小”的结果赋值给“准星扩散当前值”

  3. 添加函数“准星扩散增加”

    • 给其添加一个float类型的输入“准星扩散增加”

    • “扩散准星当前值”+输入的“准星扩散增加”

    • 判断值是否>=”准星扩散最大值”

      • true则将准星扩散当前值设置为“准星扩散最大值”

      • false则将“扩散准星当前值”+输入的“准星扩散增加”的结果赋值给“准星扩散当前值”

  4. 调用“准星扩散减小”

    • 在角色蓝图中的事件图表中的“事件Tick”后面的序列节点，添加一个引脚

    • 判断“准星扩散当前值”是否大于等于“准星扩散最小值”，true则调用“准星扩散减小”

    • 拉出事件Tick的Delta Seconds引脚，乘以“准星扩散速度”之后传入“准星扩散减小”函数的引脚

  5. 在“游戏中界面”的设计器中添加动态准星

    • 添加一个画布面板“准星十字”

    • 给其添加一个图片“上”（十字当中的最上面那一笔），将其锚点居中，调整到屏幕合适位置，尺寸修改为条形，添加十字准星的条形纹理图片

    • 同理，创建出“下，左，右”其他三个横条/竖条，合理布局为一个准星的样子

  6. 实现准星动态扩散效果

    • 在游戏中界面的图表中通过“获取类的所有actor”和”BP_Player”通信并将返回值提升为变量“蓝图角色实例”

    • “事件Tick”判断“蓝图角色实例”是否有效，有效则使用序列，分别通过“设置渲染平移”达到使得“上下左右”分别移动的效果

    • 获取”角色蓝图实例“的“准星扩散当前值”，使用“已限制映射范围”将“准星扩散最大值”引入“In Range B”

    • 设置Out Range B为“准星扩散最大值”乘以-200

    • 将计算结果设置给上下左右的“设置渲染平移”

      • 上：直接将结果设置给Y轴平移

      • 下：将结果取绝对值之后设置给Y轴平移

      • 左：同“上”连接到X轴

      • 右：同“下”连接到X轴

  7. 射击时准星持续扩散

    • 在武器父类中通过角色蓝图实例调用“准星扩散增加”

  8. 跑步时准星扩散

    • 在角色蓝图事件Tick后面的序列节点中调用“准星扩散增加”

    • 传入值根据角色速度来计算

      • “获取速度”，“向量长度”，“已限制映射范围”（400，600；0，0.01）
