使用模板函数实现切换UI界面

知识点

USizeBox

在蓝图中是一个尺寸框，当中可以存放其他组件，类似Border

在C++中

  1. USizeBox*类型的参数可以通过->GetChildAt(index)判断当前USizeBox中的下标为index的位置是否含有元素，有，则返回当前元素，无则返回空指针

  2. USizeBox*类型的参数可以通过->GetChildAt(index)->IsA()判断当前USizeBox中的下标为index的位置是否含有元素，且元素是否为IsA()中括号内的元素，是则返回true，否则返回false

  3. USizeBox*类型的参数可以通过->ClearChildren()清除体内所有元素

大致步骤

  1. 在UI_MainHall.h中

    • 添加在蓝图中已经存在的组件class UBorder* MainBoard以及class USizeBox* BoxList

  2. UI_MainHall.cpp中

    • 创建一个模板函数，用于切换UI界面，在其他界面想要切换的时候调用此模板即可

      • 确定模板函数的两个类型参数class T，class UserObject

      • 确定模板函数的返回值UserObject*

      • 确定模板函数的函数名CreateAssistWidget

      • 确定模板函数的形参列表(T* ThisClass, UClass* AssistClass, USizeBox* WidgetArray)

        • T* ThisClass代表当前类

        • UClass* AssistClass代表创建面板的类

        • USizeBox* WidgetArray代表创建到哪个位置

      • 函数体中

        • 创建一个UserObject*类型的变量UserObjectElement作为返回值，并且一开始赋值为Nullptr（返回的UI界面）

        • 使用if()判断动画（此处先不写，if中为0即可）

        • 再使用if()判断WidgetArray中是否存在元素

        • 是，则判断当中的元素是否为AssistClass

        • 是，则返回当前UI界面UserObjectElement

        • 否：如果当前WidgetArray中的元素和传入进来的WidgetArray参数不同，则删除当前WidgetArray的元素

        • 使用CreateWidget<>()创建当前UI界面UserObjectElement

          • UserObjectElement = CreateWidget<UserObject>(ThisClass->GetWorld(), AssistClass);

        • 判断UI界面是否创建成功

        • 是，则判断将UserObjectElement作为WidgetArray的子项添加进去是否成功

          • if (WidgetArray->AddChild(UserObjectElement))

        • 如果不成功，则移除UserObjectElement

        • 最后函数返回UserObjectElement

Slate简单讲解

Slate是什么？

Slate是UE4的底层UI框架。我们经常使用的UMG和UE4编辑器的各个界面底层都是Slate。

SWidget类

SWidget是所有Slate控件的基类

  • SCompoundWidget可以放一个子控件
  • SLeafWidget不能放子控件
  • SPanel可以放多个子控件

Slate的组成

在以上两者之间的声明参数，常用的参数类型有 

  • SLATE_ATTRIBUTE(属性)

  • SLATE_EVENT(事件)

  • SLATE_ARGUMENT(参数)

  • SLATE_NAMED_SLOT(插槽) 和 SLATE_DEFAULT_SLOT

创建插件

知识点

FLoadingScreenAttributes

一种数据类型，Loading界面的所有属性值都放在这里面了，声明一个此类型的变量，设置其中具体值，再赋值给SetupLoadingScreen()函数即可

  1. bAutoCompleteWhenLoadingCompletes

    • 如果为真，则在播放所有电影并完成加载后，加载屏幕将立即消失,此时进入游戏

  2. bMoviesAreSkippable

    • 如果为true，则只要加载完成，就可以通过单击加载屏幕跳过电影

  3. bWaitForManualStop

    • 如果为true，播放的动画（幻灯片）会一直循环，直到关卡加载好

  4. PlaybackType

    • 播放类型，表示我们是否应该循环播放等

  5. WidgetLoadingScreen

    • 要显示在电影顶部的小部件，如果没有电影，则只需单独显示

    • 使用SNew传入一个slate并且赋值给WidgetLoadingScreen

EMoviePlaybackType

定义了动画渲染模式，是循环播放，还是仅仅播放一次

  1. MT_Normal

    • 正常播放模式。播放播放列表中的每部电影一次

  2. MT_Looped

    • 循环播放模式。按顺序播放播放列表中的所有电影，然后重新开始，直到手动取消

  3. MT_LoadingLoop

    • 交替循环模式。播放播放列表中的所有电影，并循环播放最后一部电影，直到加载完成

GetMoviePlayer()

获取动画播放器

  1. GetMoviePlayer()->SetupLoadingScreen()

    • 设置加载界面，传入FLoadingScreenAttributes类型的参数

  2. GetMoviePlayer()->PlayMovie()

Lambda表达式/匿名函数

  1. [外部变量方位方式说明符]
    [ ] 方括号用于向编译器表明当前是一个 lambda 表达式，其不能被省略。在方括号内部，可以注明当前 lambda 函数的函数体中可以使用哪些“外部变量”。

所谓外部变量，指的是和当前 lambda 表达式位于同一作用域内的所有局部变量。

  1. (参数)
    和普通函数的定义一样，lambda 匿名函数也可以接收外部传递的多个参数。和普通函数不同的是，如果不需要传递参数，可以连同 () 小括号一起省略；

3) mutable
此关键字可以省略，如果使用则之前的 () 小括号将不能省略（参数个数可以为 0）。默认情况下，对于以值传递方式引入的外部变量，不允许在 lambda 表达式内部修改它们的值（可以理解为这部分变量都是 const 常量）。而如果想修改它们，就必须使用 mutable 关键字。

注意，对于以值传递方式引入的外部变量，lambda 表达式修改的是拷贝的那一份，并不会修改真正的外部变量；

  1. noexcept/throw()
    可以省略，如果使用，在之前的 () 小括号将不能省略（参数个数可以为 0）。默认情况下，lambda 函数的函数体中可以抛出任何类型的异常。而标注 noexcept 关键字，则表示函数体内不会抛出任何异常；使用 throw() 可以指定 lambda 函数内部可以抛出的异常类型。

值得一提的是，如果 lambda 函数标有 noexcept 而函数体内抛出了异常，又或者使用 throw() 限定了异常类型而函数体内抛出了非指定类型的异常，这些异常无法使用 try-catch 捕获，会导致程序执行失败（本节后续会给出实例）。

  1. -> 返回值类型
    指明 lambda 匿名函数的返回值类型。值得一提的是，如果 lambda 函数体内只有一个 return 语句，或者该函数返回 void，则编译器可以自行推断出返回值类型，此情况下可以直接省略-> 返回值类型。

  2. 函数体
    和普通函数一样，lambda 匿名函数包含的内部代码都放置在函数体中。该函数体内除了可以使用指定传递进来的参数之外，还可以使用指定的外部变量以及全局范围内的所有全局变量。

外部变量格式 功能
[] 空方括号表示当前 lambda 匿名函数中不导入任何外部变量。
[=] 只有一个 = 等号，表示以值传递的方式导入所有外部变量；
[&] 只有一个 & 符号，表示以引用传递的方式导入所有外部变量；
[val1,val2,…] 表示以值传递的方式导入 val1、val2 等指定的外部变量，同时多个变量之间没有先后次序；
[&val1,&val2,…] 表示以引用传递的方式导入 val1、val2等指定的外部变量，多个变量之间没有前后次序；
[val,&val2,…] 以上 2 种方式还可以混合使用，变量之间没有前后次序。
[=,&val1,…] 表示除 val1 以引用传递的方式导入外，其它外部变量都以值传递的方式导入。
[this] 表示以值传递的方式导入当前的 this 指针。

 注意，单个外部变量不允许以相同的传递方式导入多次。例如 [=，val1] 中，val1 先后被以值传递的方式导入了 2 次，这是非法的。

ConstructorHelpers::FClassFinder<>()

用于获取某个蓝图的类型class

在使用它们的时候还得遵守如下规则：

  • 只能在类的构造函数中使用，如果在普通的逻辑代码中嵌套这份代码，会引起整个编译器的crash。（实际上里面代码就有检查是否在构造函数里，否则crash）

  • 其次，FObjectFinder/FClassFinder变量必须是static的，从而保证只有一份资源实例。

  1. **FObjectFinder**：一般用来加载非蓝图资源，比如StaticMesh、Material、SoundWave、ParticlesSystem、AnimSequence、SkeletalMesh等资源：
  1. **FClassFinder**：一般用来加载蓝图资源并获取蓝图Class。这是因为如果C++要用蓝图创建对象，必须先获取蓝图的Class，然后再通过Class生成蓝图对象：
  • FClassFinder的模版名不能直接写UBlueprint，例如：FClassFinder是错误的。创建蓝图时选择的是什么父类，则写对应的父类名，假如是Actor，那么要写成：FClassFinder，否则无法加载成功。
  • FClassFinder的模版名必须和TSubclassOf变量的模版名一致，当然也可使用UClass代替TSubclassOf。实际上TSubclassOf也是UClass，只是更加强调这个Class是从T派生出来的。
  • 在启动游戏时若报错提示找不到文件而崩溃（例如：Default property warnings and errors:Error: COD Constructor (MyGameMode): Failed to find /Game/MyProject/MyBP.MyBP）
    这是因为UE4资源路径的一个规范问题，解决办法有两种：
    1. 在copy reference出来的文件路径后面加_C，例如：”Blueprint’/Game/Blueprints/MyBP.MyBP_C’”（_C可以理解为获取Class的意思）。
    2. 去掉路径前缀，例如：”/Game/Blueprints/MyBP”

封装屏幕移动功能到插件

效果：鼠标移动到屏幕边缘就可以造成屏幕相应方向的移动

知识点

UE的屏幕坐标

UE4的屏幕默认左上角为(0,0)

IsNearlyEqual()

源码

bool IsNearlyEqual(float A, float B, float ErrorTolerance = SMALL_NUMBER)

含义

A-B，如果值在ErrorTolerance之间，则返回true

注意事项

处于FMath命名空间中，在之前需要先FMath::

例如：

PlayerController->GetViewportSize()

源码

含义

获取屏幕大小，并将其X，Y值传给SizeX和SizeX

注意

必须由PlayerController->调用

PlayerController->GetMousePosition()

源码

含义

获取鼠标位置并将其传给参数LocationX和LocationY

注意

依旧需要PlayerController->调用

bShowMouseCursor

含义

为true则显示鼠标；为false则不显示鼠标

AddActorWorldOffset()

源码

含义

根据输入的偏移值（FVector类型）来使得物体进行位置偏移

FInputModeGameAndUI

一种数据结构，可以设置游戏和UI的输入模式

SetInputMode()

设置输入模式，其参数可以为FInputModeGameAndUI类型的变量

思路

  1. 在屏幕设置一个长方形的边界，在此长方形之内，鼠标随意移动，一旦超出边界，则屏幕往相应位置移动

  1. 打开TowerDefence\Plugins\StoneToolClass\Source\StoneToolClass\Public\Tool\文件夹，创建ScreenMove.h,并且在相应的Private\Tool\文件夹下面创建ScreenMove.cpp文件

  2. 在ScreenMove.h引入必要的头文件

    • 添加一个枚举EScreenMoveUnits，用于标记鼠标的位置状态

    • 声明APlayerController类

    • 创建一个结构体FScreenMoveUnits,在其中声明一个以EScreenMoveUnits为返回值的函数CursorMove，形参是APlayerController的实例，并且在.CPP文件中创建该函数的定义

  3. 在ScreenMove.cpp中定义CursorMove函数

    • 判断PlayerController是否有效，如果为true，则

    • 定义屏幕尺寸SizeX和SizeY，初始值赋值为INDEX_NONE

    • 定义鼠标位置MousePostionX和MousePostionY，赋值为-107374176.f

    • 通过APlayerController->GetViewportSize()获取屏幕尺寸并且传参给SizeX和SizeY

    • 通过PlayerController->GetMousePosition()获取鼠标位置并且传参给MousePostionX，MousePostionY

    • 用if()来限定MousePostionX，MousePostionX的取值范围（<=SizeX和SizeY&&>=0）

    • 通过FMath::IsNearlyEqual()来修改EScreenMoveUnits的状态，分别修改为上下左右，左上左下，右上右下；如果以上状态均不是，则返回为Screen_None

  4. 在ScreenMove.h中声明MoveDirection函数实现目标的移动

    • 返回值为bool

    • 形参为APlayerController* PlayerController和EScreenMoveState ScreenMoveState以及const float &ScreenMoveSpeed = 100.f

  5. 在ScreenMove.cpp中定义MoveDirection函数

    • 声明FVector类型的变量OffsetValue，赋值为FVector::ZeroVector

    • 判断PlayerController是否存在且PlayerController->GetPawn()

    • 通过switch()遍历ScreenMoveState

    • 通过PlayerController调用GetPawn()再调用AddActorWorldOffset()将偏移值OffsetValue传入其中

    • 返回值则直接为return OffsetValue!=FVector::ZeroVector;

  6. 在ScreenMove.h中声明ListenScreenMove函数实现监听屏幕是否移动成功

    • 返回值为bool

    • 形参为APlayerController* PlayerController和const float &ScreenMoveSpeed=100.f

  7. 在ScreenMove.cpp中定义ListenScreenMove函数

    • 直接返回函数MoveDirection

  8. 在TowerDefence.build.cs文件中添加StoneToolClass模块

    • 打开TowerDefence.build.cs，在PublicDependencyModuleNames.AddRange(new string[] { "Core", "CoreUObject", "Engine", "InputCore" });的最后，添加,"StoneToolClass"

  9. 在TowerDefencePlayerController.h文件中添加#include "Tool/ScreenMove.h"

    • 并且在protect下创建一个FScreenMoveUnits结构体的实例ScreenMoveUnits

  10. 在TowerDefencePlayerController.cpp文件中调用

    • 在Tick时间中声明float类型的变量ScreenMoveSpeed并且赋值为100.f

    • 调用ScreenMoveUnits实例中的函数ListenScreenMove()

  11. 在ScreenMove.h中的结构体FScreenMoveUnits前面添加一个导出宏STONETOOLCLASS_API

  12. 在ScreenMove.cpp文件中的MoveDirection()函数中使用switch()遍历ScreenMoveState

    • 每遍历到一个状态值的时候使用OffsetValue=FVector(ScreenMoveSpeed,0.0f,0.0f)来给OffsetValue赋值

    • 在遍历完之后使用PlayerController->GetPawn()->AddActorWorldOffset(OffsetValue);使得玩家控制器偏移

  13. 在TowerDefencePlayerController.cpp中

    • 将ScreenMoveSpeed的值修改为20.0f

    • 在构造函数中设置显示鼠标bShowMouseCursor=true;

  14. 在TowerDefencePlayerController.h中添加函数SetInputModeGameAndUI()

    • 在cpp文件中定义

    • 添加一个FInputModeGameAndUI类型的变量InputMode

    • 设置鼠标一直锁定在视口范围内InputMode.SetLockMouseToViewportBehavior(EMouseLockMode::LockAlways)

    • 关闭按下鼠标则鼠标隐藏功能InputMode.SetHideCursorDuringCapture(false);
