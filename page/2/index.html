ScriptClass

常见三种类型的Script

  1. Script

  2. LocalScript

  3. ModuleScript

Script

定义

Script是一种Lua代码容器，它将在服务器上运行其内容。

满足一定条件之后，Script中的代码会在一个新的线程中运行

运行条件

只有Script满足以下两个条件的时候，其中的代码才会运行

  1. 不被禁用（Disable属性值为false）

  2. 在WorkSpace或者ServerScriptService下

  3. 如果不像其立刻运行，则将其存放在ServerStorage下

    • ServerStorage：专门用于存放预先制作好的对象，而此对象只会在调用时运行；只存在于服务端，客户端不可用且LocalScript不可访问；仅能通过DataModel.GetService进行访问

终止条件

  1. 不满足运行条件

  2. 终止或者引发错误

    • 如果该错误是连接到正在激发的某个事件的函数引发的，则此Script不会终止

  3. 此Script或者其某一个父类被destroy的时候，该线程也会停止

  4. 在Script的父类没有被destroy的时候，即使其父类为nil，此Script也会继续运行

特性

Script可以访问服务器端对象、属性和事件

例如：

Script可以使用BadgeService向玩家颁发徽章，而客户端上的LocalScript则不能。

LocalScripts执行的未复制的操作（由于Workspace.FiltingEnabled）对脚本不可见。

Archivable    (boolean)

此属性确定在发布/保存游戏时，或在对象的某个父类上调用Instance:Clone（）时，是否应包含对象。

如果被克隆的对象Archivable==false，则Instance:Clone()将会return false。

在Studio中复制对象（使用“Duplicate”或“Copy”选项）将忽略Archivible属性，并将副本的Archivible设置为true。

ClassName (string)

  1. 表示此Instance所属类的只读字符串，不能由脚本更改。更改Instance类将会创建一个新的Instance

  2. 此属性可以与实例的各种其他函数一起使用，这些函数 用于按类型标识对象，例如Instance:IsA()或Instance:FindFirstChildOfClass()

  3. 和Instance:IsA()不同，ClassName可以用于检查对象是否属于忽略类继承的特定类。【used to check if an object belongs to a specific class ignoring class inheritance.】

实例的非唯一标识符。
此属性是描述对象的标识符。一个对象的多个子对象可以共享相同的名称。名称用于组织对象层次结构，同时允许脚本访问特定对象。

Parent

Parent属性确定实例的层次父级

Remove函数将此属性设置为nil。调用Destroy将将实例的父级及其所有子级设置为零，并锁定父级属性。设置已销毁对象的父级时将会引发错误。

此属性还用于管理游戏中是否存在对象或是否需要删除对象。只要对象A的父对象位于DataModel中、存储在变量中或被另一个对象属性引用，则该对象将保留在游戏中。否则，对象将自动删除。顶级DataModel对象（脚本称之为游戏）没有父对象，但始终有一个由游戏引擎保存的引用，并在会话期间存在。

可以使用Instance.new()创建一个新的实例（需要指定Parent）

服务端/客户端复制

服务器创建的对象将不会复制到客户端，除非它是某个已复制对象的父对象。当创建一个对象并设置许多属性时，建议最后设置“父级”。这样可以确保对象复制一次，而不是复制许多属性更改。

Disabled

确定BaseScript是否运行。
当Disabled==false时，Scripts和LocalScripts将运行。
如果在脚本运行时将Disabled设置为true，则当前线程将终止。
如果Disabled从false设置为true，脚本将再次运行。

这意味着可以切换Disabled属性以重新启动脚本

LocalScript

LocalScript是一个Lua源容器，专用于客户端代码，例如玩家的相机，或者各种UI

对于通过LocalScripts运行的代码，玩家服务的LocalPlayer属性将返回其客户端正在运行此脚本的玩家。

LocalScript仅会作为以下东西的子类时候才会运行

  1. 玩家的Backpack，例如Tool的子类

  2. 玩家的Character模型

  3. 玩家的PlayerGui

  4. 玩家的PlayerScripts

  5. ReplicatedFirst

后面的属性与方法，事件与Script差不多，这里就不再赘述

ModuleScript

  1. ModuleScripts在每个Lua环境中仅运行一次；如果后续还要调用此脚本，则会返回完全相同的值。

  2. ModuleScripts的返回值与LocalScripts和Scripts以及其他环境（如命令栏）无关。

  3. 如果ModuleScriptA运行没有结束，FunctionA调用ModuleScriptA也不会结束，除非ModuleScriptA运行完

    • 例如：如果ModuleScriptA尝试调用ModuleScriptB，此时ModuleScriptB又尝试调用ModuleScriptA，则会出现死循环

  4. 在LocalScript中对ModuleScript使用require将在客户端上运行代码，即使脚本已经在服务器上运行。

    • 因此，如果同时在客户端和服务器上使用ModuleScript，或者在Studio中调试它，请务必小心。

  5. 适合在里面写函数，需要的时候进行调用即可

unLua-释放UMG相关对象

发表于 2022-10-20 分类于 unLua

UMG对象的释放流程

  1. 调用self:Release()，解除LuaTable在C++侧的引用

  2. 确保LuaTable在Lua侧没有其他引用，触发LuaGC

  3. C++侧收到UObject_Delete回调，解除UMG在C++侧的引用

  4. 确保UMG在C++侧没有其他引用，触发UEGC

注意

使用控制台命令查看对象和类的引用情况：

查看指定类的引用列表：Obj List Class=ReleaseUMG_Root_C

查看指定对象的引用链：Obj Refs Name=ReleaseUMG_Root_C_0
